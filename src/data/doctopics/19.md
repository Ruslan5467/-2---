# Рекурсивные алгоритмы на примере задачи о расстановке ферзей

## Теоретическая база: Перебор с возвратом (Backtracking)

Ранее мы рассматривали рекурсию как математический инструмент (вычисление факториала или чисел Фибоначчи). Однако в системной инженерии и алгоритмике рекурсия является фундаментальным механизмом для реализации мощного паттерна поиска решений — **Перебора с возвратом (Backtracking)**.

**Бэктрекинг** — это систематический метод перебора всех возможных вариантов решения задачи. Его главное отличие от "грубой силы" (Brute-force) заключается в интеллектуальном отсечении бесперспективных путей.
Алгоритм шаг за шагом строит частичное решение. Если на каком-то шаге возникает противоречие (становится понятно, что текущий частичный путь гарантированно зайдет в тупик), алгоритм отменяет последний сделанный шаг (совершает _возврат_ аппаратно через раскрутку стека рекурсии) и пробует следующую доступную альтернативу.

Именно использование стека вызовов (рекурсии) делает реализацию бэктрекинга элегантной: нам не нужно вручную программировать сложные механизмы "памяти прошлых шагов" — операционная система делает это за нас, сохраняя локальные переменные на каждом уровне рекурсивного погружения.

## Постановка задачи и Инженерная модель (N-Queens)

**Формулировка:** Дана стандартная шахматная доска размером N×N. Необходимо расставить на ней N ферзей таким образом, чтобы ни один из них не находился под боем другого. (Ферзь бьет по вертикали, горизонтали и обеим диагоналям).

**Алгоритмическая модель (Оптимизация представления):**
Наивный подход предложил бы создать двумерный массив `board[N][N]` и пытаться ставить ферзей. Но профессиональная инженерия требует оптимизации.
Так как по правилам шахмат на одной вертикали не может стоять больше одного ферзя, мы можем значительно сузить пространство поиска: мы будем гарантированно ставить ровно по одному ферзю в каждый столбец (от 0 до N-1).

Мы можем хранить состояние доски в простом **одномерном массиве** `int queens[N]`, где индекс элемента — это номер столбца, а значение элемента — номер строки, в которой стоит ферзь в этом столбце. Это сокращает потребление памяти и ускоряет проверки.

## Пример использования (Реализация на C++)

Алгоритм пытается поставить ферзя в текущий столбец (начиная с 0). Он перебирает все строки (от 0 до N-1) в этом столбце, проверяет безопасность позиции и, если успешно, — рекурсивно вызывает себя для следующего столбца.

```cpp
#include <iostream>
#include <vector>
#include <cmath>

// Функция проверки безопасности (Бьют ли уже поставленные ферзи текущую клетку)
// queens - хранит строки ферзей для столбцов [0 ... col-1]
bool isSafe(const std::vector<int>& queens, int tryRow, int col) {
    for (int prevCol = 0; prevCol < col; ++prevCol) {
        int prevRow = queens[prevCol];

        // 1. Проверка горизонтали (на одной ли строке?)
        if (prevRow == tryRow) return false;

        // 2. Проверка диагоналей. Дистанция по X равна дистанции по Y
        if (std::abs(prevRow - tryRow) == std::abs(prevCol - col)) return false;
    }
    return true; // Позиция безопасна
}

// Рекурсивный алгоритм бэктрекинга
// col - столбец, в который мы сейчас пытаемся поставить ферзя
void solveNQueens(std::vector<int>& queens, int col, int N, int& solutionsCount) {
    // 1. БАЗОВЫЙ СЛУЧАЙ (Терминальное условие)
    // Если мы дошли до столбца, равного N (за пределами доски),
    // значит мы успешно поставили всех N ферзей!
    if (col == N) {
        solutionsCount++;
        // В реальном коде здесь можно было бы напечатать массив queens
        return;
    }

    // 2. РЕКУРСИВНЫЙ ШАГ (Поиск с возвратом)
    // Перебираем все возможные строки в текущем столбце
    for (int row = 0; row < N; ++row) {
        if (isSafe(queens, row, col)) {
            // ДЕЛАЕМ ШАГ: Ставим ферзя
            queens[col] = row;

            // РЕКУРСИЯ: Пытаемся расставить оставшихся ферзей в следующих столбцах
            solveNQueens(queens, col + 1, N, solutionsCount);

            // ВОЗВРАТ (Backtrack): Стираем ферзя (технически, значение перезапишется
            // на следующей итерации цикла, но концептуально это шаг назад)
        }
    }
}

int main() {
    int N = 8; // Классическая задача для доски 8х8
    std::vector<int> queens(N, 0); // Вектор состояний
    int solutionsCount = 0;

    solveNQueens(queens, 0, N, solutionsCount);

    std::cout << "Для доски " << N << "x" << N << " найдено ";
    std::cout << solutionsCount << " вариантов расстановки.\n";
    // Для N=8 ответ в математике известен: 92 классических решения
    return 0;
}
```

## Протокол прохождения (Дерево вариантов и Отсечение ветвей)

Проследим логику работы АЛУ процессора на примере маленькой доски 4x4 (`N=4`).

1.  **Старт (`col = 0`):** Мы в начале пути. Пытаемся поставить первого ферзя в столбец 0. Цикл `row` начинается с 0.
    - Ставим ферзя в `(0,0)`. Доска: `[0, X, X, X]`.
    - Рекурсивный вызов: `solveNQueens(..., col = 1)`.
2.  **Глубина 1 (`col = 1`):** Ищем место для второго ферзя.
    - `row = 0`: Конфликт по горизонтали. `isSafe` = false. Отсекаем (Prune).
    - `row = 1`: Конфликт по диагонали. `isSafe` = false. Отсекаем.
    - `row = 2`: Безопасно! Доска: `[0, 2, X, X]`.
    - Рекурсивный вызов: `solveNQueens(..., col = 2)`.
3.  **Глубина 2 (`col = 2`):** Ищем место для третьего ферзя.
    - `row = 0, 1, 2, 3` — проверка `isSafe` покажет конфликты для всех четырех клеток!
    - Цикл `for` на глубине 2 завершается, не сделав ни одного успешного шага.
4.  **ВОЗВРАТ (Backtracking):** Фрейм `col=2` уничтожается. Процессор "всплывает" обратно во фрейм `col=1`.
    - Процессор вспоминает: "Ага, в столбце 1 я остановился на `row=2`. Что там дальше?".
    - Он пробует `row = 3`: Безопасно! Доска: `[0, 3, X, X]`.
    - Снова рекурсивный вызов: `solveNQueens(..., col = 2)`...
5.  И так далее, пока алгоритм не прощупает все дерево комбинаций.

Это поведение похоже на прохождение лабиринта. Если вы уперлись в тупик на 3-м шаге, вы не начинаете лабиринт с самого начала. Вы делаете шаг назад на перекресток (2-й шаг) и выбираете другой коридор. Стек вызовов языка C++ автоматически служит для системы "клубком ниток", который позволяет безошибочно возвращаться на предыдущие перекрестки.

## Резюме характеристик

| Характеристика        | Описание                                                                                                                                    |
| :-------------------- | :------------------------------------------------------------------------------------------------------------------------------------------ |
| **Парадигма**         | Backtracking (Перебор с возвратом через рекурсию)                                                                                           |
| **Механизм возврата** | Естественное свойство стека вызовов (схлопывание фрейма возвращает локальные циклы к предыдущим состояниям)                                 |
| **Оптимизация**       | Раннее отсечение ветвей (Pruning) с помощью проверочной функции (в нашем случае `isSafe`). Блокирует развитие заведомо тупиковых вариантов. |
| **Применение класса** | Решение шахматных задач, взлом паролей, генерация судоку, выход из лабиринта, задача коммивояжера.                                          |
