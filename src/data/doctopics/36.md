# Двоичное дерево поиска (Binary Search Tree - BST)

## Теоретическая база: Что такое Бинарное Дерево Поиска?

Обычное бинарное дерево (где у каждого узла максимум два потомка) само по себе не дает никаких структурных преимуществ для быстрого поиска. Если числа раскиданы по дереву в хаотичном порядке, то для поиска конкретного числа нам придется обойти всё дерево целиком ($O(N)$), что ничем не лучше обычного непорядоченного массива или связного списка.

Чтобы деревья стали по-настоящему полезными поисковыми структурами, на их архитектуру накладывают жесткое математическое правило сортировки. Такое дерево называют **Бинарным (Двоичным) Деревом Поиска** (Binary Search Tree — BST).

**Свойства (Правила) BST:**

1.  **Левое поддерево:** Значения ВСЕХ узлов в левом поддереве должны быть строго **МЕНЬШЕ**, чем значение родительского узла.
2.  **Правое поддерево:** Значения ВСЕХ узлов в правом поддереве должны быть строго **БОЛЬШЕ**, чем значение родительского узла.
3.  **Рекурсивность:** Оба поддерева (левое и правое) сами по себе тоже должны являться идеальными бинарными деревьями поиска.
4.  **Уникальность (Опционально):** В классическом строгом BST не допускается наличие дубликатов. (Если дубликаты разрешены, то правило "равно" прикрепляется к одной из сторон, например, $\le$ слева).

### Как это ускоряет поиск?

Поиск в BST работает по принципу интеллектуального отсечения половины вариантов на каждом шаге (аналог бинарного поиска `std::binary_search` в отсортированном массиве).

Представьте, что мы ищем число `30`:

1.  Мы стоим на корне `50`. Спрашиваем: `30` больше или меньше `50`? Меньше! Значит, мы **навсегда отбрасываем всю правую ветвь** (и все тысячи лежащих там чисел) и уходим налево.
2.  Слева корень `25`. `30` больше `25`. Отбрасываем левую ветвь и уходим направо.
3.  Находим `30`.

В идеально сбалансированном дереве каждый шаг вниз отсекает половину оставшихся узлов.

## Временная сложность BST

Архитектурная эффективность бинарного дерева поиска напрямую зависит от его Высоты (глубины слоев).

| Случай                         | Временная сложность (Поиск, Вставка, Удаление) | Описание                                                                                                                                                                                                                                                                                                                                                 |
| :----------------------------- | :--------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Средний (идеальный) случай** | $O(\log N)$                                    | Дерево структурно сбалансировано (похоже на пирамиду). Высота дерева равна логарифму по основанию $2$ от числа узлов. Для поиска среди $1~000~000$ узлов потребуется не более $20$ шагов (сравнений)!                                                                                                                                                    |
| **Худший случай**              | $O(N)$                                         | "Вырождение дерева" (Degenerate Tree). Если мы добавляем в пустое BST уже отсортированные данные (например, `1`, `2`, `3`, `4`), дерево превращается в линейный односвязный список, растущий только вправо. Поиск становится медленным (линейным). Для защиты от этого существуют алгоритмы "самобалансирующихся деревьев" (АВЛ, Красно-черные деревья). |

## Реализация BST на C++ (Вставка и Поиск)

Для поддержания внутренней организации BST, мы не имеем права строить ветви "как попало" руками в `main`. Процесс создания и сборки Дерева Поиска обязан осуществляться через специальную функцию `insertBST`, которая сама найдет правильное место новому узлу.

```cpp
#include <iostream>

struct TreeNode {
    int data;
    TreeNode* left;
    TreeNode* right;

    TreeNode(int val) : data(val), left(nullptr), right(nullptr) {}
};

// 1. ФУНКЦИЯ ВСТАВКИ В BST (Рекурсивная)
// Возвращает указатель на корень поддерева
TreeNode* insertBST(TreeNode* node, int value) {
    // Если дошли до тупика (nullptr), значит мы нашли место для нового узла!
    if (node == nullptr) {
        return new TreeNode(value);
    }

    // Если значение меньше родителя — спускаемся (делегируем задачу) в ЛЕВУЮ ветку
    if (value < node->data) {
        node->left = insertBST(node->left, value);
    }
    // Если значение больше родителя — спускаемся в ПРАВУЮ ветку
    else if (value > node->data) {
        node->right = insertBST(node->right, value);
    }

    // Если value == node->data, мы просто игнорируем его (дубликаты запрещены)

    // Возвращаем неизменный текущий узел обратно рекурсии
    return node;
}

// 2. ФУНКЦИЯ ПОИСКА В BST (Рекурсивная)
// Возвращает указатель на найденный узел, либо nullptr, если числа нет
TreeNode* searchBST(TreeNode* node, int target) {
    // Базовый случай 1: дошли до конца (число не найдено), либо дерево пустое
    if (node == nullptr) return nullptr;

    // Базовый случай 2: Бинго! Нашли число!
    if (node->data == target) return node;

    // Рекурсивный спуск с отсечением ненужной ветви
    if (target < node->data) {
        return searchBST(node->left, target); // Уходим налево
    } else {
        return searchBST(node->right, target); // Уходим направо
    }
}

// Симметричный обход для проверки порядка алгоритмом L-N-R
void inOrder(TreeNode* node) {
    if (node == nullptr) return;
    inOrder(node->left);
    std::cout << node->data << " ";
    inOrder(node->right);
}

// Очистка памяти
void deleteTree(TreeNode* node) {
    if (node == nullptr) return;
    deleteTree(node->left);
    deleteTree(node->right);
    delete node;
}

int main() {
    TreeNode* root = nullptr;

    // Вставляем элементы вразнобой
    root = insertBST(root, 50); // Первый добавленный элемент становится корнем
    root = insertBST(root, 25);
    root = insertBST(root, 75);
    root = insertBST(root, 10);
    root = insertBST(root, 30);
    root = insertBST(root, 90);

    // Доказательство сортировки:
    std::cout << "Элементы дерева (in-order обход): ";
    inOrder(root); // Выведет: 10 25 30 50 75 90
    std::cout << "\n";

    // Поиск значения с отсечением лишних ветвей (за O(log N))
    int target = 30;
    TreeNode* result = searchBST(root, target);

    if (result != nullptr) {
        std::cout << "Успех! Узел " << target << " найден в дереве!\n";
    } else {
        std::cout << "Ошибка! Узел " << target << " не найден.\n";
    }

    deleteTree(root);
    return 0;
}
```

## Протокол прохождения (Механика вставки `insertBST`)

Как число `30` нашло свое место в коде алгоритма?

1. Мы вызываем `root = insertBST(root(50), 30)`.
2. В узле `50` срабатывает `if (30 < 50)`. Программа командует: совершить рекурсивную вставку в левое поддерево (`node->left = insertBST(...)`). Мы уходим влево.
3. В узле `25` срабатывает `if (30 > 25)`. Мы уходим вправо, в пустоту (`node->right = insertBST(nullptr, 30)`).
4. В пустоте срабатывает `if (node == nullptr)`. Этот участок кода создает физический узел `new TreeNode(30)` в Куче и возвращает его адрес назад.
5. Возврат всплывает вверх по Call Stack: узел `25` ловит в `node->right` адрес новой тридцатки (мост построен!). Возврат идет к корню. Корень ловит в `node->left` свой же старый адрес 25 (мост переподтвержден).

## Архитектурные трудности (Удаление из BST)

Удалить элемент из середины BST гораздо сложнее, чем вставить. Если вы удаляете **Корень** или узел, у которого есть оба поддерева (левое и правое), вы не можете просто "стереть" его — дерево развалится на два независимых куска, а родитель жертвы может прицепить к себе "на усыновление" только одного ребенка.

Для этой ситуации существует "Алгоритм замены":

1. Найти узел-жертву.
2. В её **правом поддереве** найти **самое маленькое** число (оно находится, спускаясь до упора влево). Это число будет ближайшим по значению к удаляемому.
3. Скопировать это самое маленькое число в узел-жертву (просто затерев и заместив старое значение).
4. Рекурсивно удалить тот самый левый нижний лист из правого поддерева (удалять листья легко, у них нет детей).

## Резюме характеристик

| Характеристика                  | Описание                                                                                                                                                                |
| :------------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Главное правило**             | Слева — строго меньше, справа — строго больше родителя.                                                                                                                 |
| **Асимптотическая сложность**   | В сбалансированном дереве $O(\log N)$ — логарифмическая.                                                                                                                |
| **Кошмарный сценарий ($O(N)$)** | Вставка в пустое дерево уже заранее отсортированных данных. Дерево вытягивается в длинную нить (вырождается в связный список). Спасение: алгоритмы с автобалансировкой. |
| **Особенность обхода**          | Применение Симметричного обхода (In-order, L-N-R) математически гарантировано выведет содержимое BST по возрастанию.                                                    |
