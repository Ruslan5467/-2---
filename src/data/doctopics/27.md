# Линейный односвязный список без фиктивного головного элемента, заполненный в обратном порядке

## Теоретическая база: Стратегия LIFO (Last-In, First-Out)

В предыдущем вопросе мы рассмотрели базовую организацию односвязного линейного списка без фиктивного головного элемента и операцию добавления в его начало (функция `pushFront`).

Если мы последовательно считываем элементы (например, из пользовательского ввода или из файла) и каждый новый элемент добавляем **в начало** списка, то мы реализуем принцип **LIFO** (Last-In, First-Out — "последним пришел, первым ушел"). Это классическое определение работы структуры данных **Стек (Stack)**.

При таком подходе элементы в списке выстраиваются **в обратном порядке** по отношению к тому порядку, в котором они к нам поступали.

Например, если мы по очереди получили числа `1`, `2`, `3`:

1. Вставляем `1`: `[1] -> nullptr`
2. Вставляем `2`: `[2] -> [1] -> nullptr`
3. Вставляем `3`: `[3] -> [2] -> [1] -> nullptr`

Итоговый список (при чтении от головы к хвосту) содержит элементы в порядке `3, 2, 1`.

---

## Архитектурное преимущество заполнения в обратном порядке

Почему такой метод добавления элементов используется в разработке очень часто? Ответ: **Производительность**.

Для вставки нового узла в самое начало односвязного списка без фиктивной логической головы нам **не требуется** знать, где находится конец списка. Мы работаем исключительно с указателем `head`. Вставка происходит мгновенно за константное время — **$O(1)$**.

Если бы мы захотели добавлять элементы в _конец_ такого списка (чтобы сохранить прямой порядок), нам бы приходилось на каждой итерации:

1. Запускать цикл от `head` и "бежать" по всем мостам `next`, пока не найдем узел с `next == nullptr`.
2. Это занимало бы $O(N)$ времени на каждую вставку, и полное построение списка заняло бы $O(N^2)$ времени. (Эту проблему прямого прохода обычно решают введением дополнительного указателя `tail` — на хвост списка).

Следовательно, "Стек" (список в обратном порядке) — это самый дешевый способ собрать элементы в связный список.

## Пример использования (Сборка списка и обход)

Разберем классический пример: сборка связного списка целых чисел, вводимых пользователем. Признаком окончания ввода будет служить, например, число `0`.

```cpp
#include <iostream>

// 1. СТРУКТУРА УЗЛА
struct Node {
    int data;       // Полезная нагрузка
    Node* next;     // Указатель-мост на следующий вагон
};

// Функция добавления в начало (LIFO)
void pushFront(Node*& head, int value) {
    Node* newNode = new Node;
    newNode->data = value;
    newNode->next = head; // Привязываем новый узел к старой голове
    head = newNode;       // Корона возвращается новому узлу
}

// Функция для печати элементов списка
void printList(Node* head) {
    std::cout << "Содержимое списка (от головы к хвосту):\n";
    Node* current = head;
    while (current != nullptr) {
        std::cout << current->data << " -> ";
        current = current->next;
    }
    std::cout << "NULL\n";
}

// Функция сборки списка из пользовательского потока (std::cin)
// Заполнение будет идти в обратном порядке! Вставляем всегда в начало.
Node* buildListReversed() {
    Node* head = nullptr; // Изначально список пуст
    int inputVal;

    std::cout << "Введите числа для добавления в список (0 - для завершения):\n";
    while (true) {
        std::cin >> inputVal;

        // 0 - Терминальное условие завершения ввода
        if (inputVal == 0) {
            break;
        }

        // Вызов функции вставки за константное время O(1)
        pushFront(head, inputVal);
    }

    // Возвращаем сформированную голову
    return head;
}

// Функция очистки (Предотвращение утечки памяти!)
void deleteList(Node*& head) {
    while (head != nullptr) {
        Node* temp = head;       // Спасаем адрес текущей головы
        head = head->next;       // Перемещаем голову списка на следующий шаг
        delete temp;             // Освобождаем память спасенного узла
    }
}

int main() {
    // Вызываем процесс формирования списка в куче (Heap)
    // Функция вернет указатель на начало сформированного LIFO-списка
    Node* myListHead = buildListReversed();

    std::cout << "\nРезультат:\n";
    printList(myListHead);

    // Удаляем списковый контейнер
    deleteList(myListHead);

    return 0;
}
```

## Протокол прохождения (Реверсивная сборка)

Предположим, оператор сидит за консолью и вводит последовательность чисел: `5`, `15`, `99`, `0`. Рассмотрим, что происходит в памяти (в Куче) на каждом этапе цикла.

- **Инициализация:** `head = nullptr;` (Пусто).
- **Итерация 1 (`inputVal = 5`):**
  - `pushFront` создает новый блок: `Node(5, nullptr)`.
  - Склеивает его с пустотой: `newNode->next = head;` (то есть `nullptr`).
  - Переопределяет голову: `head` указывает на блок `[5]`.
  - _Вид в памяти:_ `[5] -> nullptr`
- **Итерация 2 (`inputVal = 15`):**
  - `pushFront` создает новый блок: `Node(15, ?)`.
  - Склеивает: `newNode->next = head;` (блок `15` теперь смотрит на блок `5`).
  - Переопределяет голову: `head` теперь указывает на блок `[15]`.
  - _Вид в памяти:_ `[15] -> [5] -> nullptr`
- **Итерация 3 (`inputVal = 99`):**
  - `pushFront` создает новый блок: `Node(99, ?)`.
  - Склеивает его со старой головой `[15]`.
  - Перемещает флаг головы `head` на `[99]`.
  - _Вид в памяти:_ `[99] -> [15] -> [5] -> nullptr`
- **Итерация 4 (`inputVal = 0`):**
  - Срабатывает условие выхода (`break`). Сборка завершена. Возвращаем `head`.

**Итог:** Мы получили элементы в обратной хронологии: самым первым элементом стал `99`, так как он вытеснил остальные в глубину связного "колодца". Это принцип классического магазинного стека.

## Резюме характеристик

| Элемент                        | Описание                                                                                      |
| :----------------------------- | :-------------------------------------------------------------------------------------------- |
| **Парадигма добавления**       | LIFO (Стек — последним пришел, первым ушел).                                                  |
| **Производительность (Время)** | Быстро и идеально — **$O(1)$** для каждой вставки. Отсутствие скрытых циклов поиска конца.    |
| **Использование памяти**       | Динамическое выделение (Heap) по требованию на каждый поступающий элемент.                    |
| **Ориентация ссылок**          | "Свежий" элемент списка всегда имеет своим полем `next` ссылку на цепочку "старых" элементов. |
