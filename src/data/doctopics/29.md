# Основные операции с односвязными линейными списками с динамическим головным элементом (фиктивной головой)

## Теоретическая база: Зачем нужен фиктивный головной элемент?

В предыдущей теме, обсуждая списки _без_ головного элемента, мы выяснили, что функции вставки в начало и удаления первого логического узла требуют специального, отдельного блока кода (`if (head == ...)`). Это происходило потому, что первый элемент списка "висел в воздухе" — у него физически не было предшественника, чей указатель `next` можно было бы перенаправить. Мы были вынуждены модифицировать сам системный указатель `head`.

Архитектурный паттерн **Списка с фиктивным головным элементом (Dummy Head / Sentinel Node)** элегантно решает эту проблему однородности алгоритма.

**Суть паттерна:**
При создании пустого списка мы сразу и принудительно выделяем в куче (Heap) один абсолютно пустой, служебный узел. И системный указатель `head` навсегда "прибивается гвоздями" именно к этому служебному узлу.
Этот узел никогда не хранит полезных данных и никогда не удаляется (пока существует сам список). Его единственная работа — просто быть, чтобы служить постоянной "лестничной площадкой" для следующего за ним первого _реального_ узла данных.

**Топология:**

- Пустой список: `[ФИКТИВНЫЙ] -> nullptr`
- Один элемент: `[ФИКТИВНЫЙ] -> [Данные_1] -> nullptr`
- Два элемента: `[ФИКТИВНЫЙ] -> [Данные_1] -> [Данные_2] -> nullptr`

**Главный профит:** Теперь у **любого** элемента с полезной нагрузкой (пользовательскими данными), даже у логически первого, математически **гарантированно существует узел-предшественник**. Это означает, что логика алгоритмов вставки и удаления становится абсолютно универсальной: исчезают все специальные блоки `if` краевых случаев!

## Инициализация структуры

Обратите внимание, что указатель `head` больше никогда не будет равен `nullptr`.

```cpp
#include <iostream>

struct Node {
    int data;
    Node* next;
};

// Функция создания нового контейнера (возвращает фиксированную фиктивную голову)
Node* createList() {
    Node* dummyHead = new Node; // Выделяем память
    dummyHead->data = 0;        // Мусор/заглушка (эти данные никогда не читаются)
    dummyHead->next = nullptr;  // Следующего логического узла пока нет
    return dummyHead;
}
```

## 1. Вставка в начало (Теперь это "Вставка после головы")

Архитектурная магия фиктивной головы заключается в том, что вставка элементов в вершину стека (`pushFront`) превращается просто в вызов "Вставить после узла `head`". Нам больше не нужно передавать указатель по ссылке (`Node*&`), потому что мы никогда не переопределяем адрес самой головы.

```cpp
// Добавление в начало реального списка
// (Константное время O(1), аргумент head передается по значению)
void insertFront(Node* head, int value) {
    if (head == nullptr) return; // Защита от дурака

    Node* newNode = new Node;
    newNode->data = value;

    // Классическая универсальная сшивка (вставляем МЕЖДУ head и первым реальным узлом)
    newNode->next = head->next;
    head->next = newNode;
}
```

## 2. Удаление первого вхождения по значению (Delete by value)

Помните тяжелый код удаления из предыдущей лекции? Теперь он становится монолитным и кристально чистым, без "особых случаев краевой вставки". Мы ставим бегунка на фиктивную голову и уверенно применяем логику "посмотри на следующий узел".

```cpp
void deleteValue(Node* head, int targetValue) {
    if (head == nullptr) return;

    Node* current = head; // Start point - это ПУСТОЙ фиктивный узел

    // Цикл не сорвется, потому что если первый РЕАЛЬНЫЙ узел хранит targetValue,
    // алгоритм отцепит его, модифицировав next фиктивной головы.
    while (current->next != nullptr && current->next->data != targetValue) {
        current = current->next;
    }

    if (current->next == nullptr) {
        return; // Значение не найдено
    }

    // Сохраняем адрес жертвы (Она гарантированно существует и имеет предшественника)
    Node* nodeToDelete = current->next;

    // Сшиваем разорванный список "через голову" жертвы
    current->next = nodeToDelete->next;

    // Уничтожаем физический блок оперативной памяти
    delete nodeToDelete;
}
```

## 3. Печать и полная очистка списка

При обходе списка с фиктивной головой мы обязаны делать пропуск первого узла.

```cpp
// Вывод на экран списка
void printList(Node* head) {
    if (head == nullptr) return;

    // Начинаем парсинг сразу со ВТОРОГО узла (с первого реального)
    Node* current = head->next;

    std::cout << "Список: ";
    while (current != nullptr) {
        std::cout << current->data << " -> ";
        current = current->next;
    }
    std::cout << "NULL\n";
}

// Очистка (не забываем удалить и саму фиктивную голову)
void destroyList(Node*& head) {
    while (head != nullptr) {
        Node* temp = head;
        head = head->next;
        delete temp;
    }
}
```

## Протокол прохождения (Механика чистого удаления логической верхушки)

Разберем, насколько гладко работает процесс удаления первого "реального" числа `15` из списка:
`[DUMMY] -> [15] -> [99] -> NULL`

1. Вызов `deleteValue(head, 15)`.
2. Указатель-бегунок `current` встает на клетку `[DUMMY]`.
3. Анализ цикла: `current->next->data` — это, по технической сути, данные _первого логического_ узла `[15]`. Они равны целевому числу `15`. Цикл `while` моментально рушится (даже не сделав ни единого шага вглубь!).
4. Изоляция жертвы: `nodeToDelete = current->next;` (спасаем адрес узла `[15]`).
5. Сшивка: `current->next = nodeToDelete->next`. Бегунок (`current`), который стоит на `[DUMMY]`, перебрасывает свой мост-указатель напрямую на узел `[99]`.
6. Дефрагментация: `delete nodeToDelete;` (узел `[15]` стирается в небытие).
7. Итоговая конфигурация списка: `[DUMMY] -> [99] -> NULL`.

Как инженеры, мы добились превосходного результата: нам не пришлось менять внешний, самый главный адрес `head`, который держала другая подсистема. Нам не пришлось писать "костыль" с `if (первый == цель)`. Алгоритм отработал по универсальной формуле.

## Резюме характеристик

| Характеристика                         | Описание                                                                                                                                                                                      |
| :------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Суть паттерна (Dummy Head)**         | Принудительное выделение служебного пустого узла (без полезных данных), который объявляется вечной, недвижимой головой списка.                                                                |
| **Главное архитектурное преимущество** | Радикальное упрощение математики и кода. Исключение проверок "Краевых случаев" (обработка пустых списков или удаления/вставки самых первых логических узлов) из тела циклов $O(N)$.           |
| **Отношение к передаче параметров**    | В подавляющем большинстве операций `head` передается "как значение" (copy_by_value), потому что сам адрес физического фиктивного контейнера `head` никогда не модифицируется и не сдвигается. |
| **Побочные расходы**                   | Незначительные 12–16 байт оперативной памяти (Heap) на один бесполезный узел-заглушку.                                                                                                        |
