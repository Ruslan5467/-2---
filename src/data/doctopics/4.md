# Динамические массивы. Двумерные динамические массивы

## Теоретическая база и архитектурная абстракция

В отличие от статических двумерных массивов (матриц), память под которые выделяется на стеке как единый, строго непрерывный блок байтов, организация двумерных массивов в динамической памяти (куче) представляет собой значительно более сложную, составную архитектуру.

Язык C++ не предоставляет встроенного оператора для прямого создания двумерной матрицы в куче одной командой (если неизвестны размеры во время компиляции). Поэтому классическая реализация двумерного динамического массива строится на концепции **«массива указателей» (Array of Pointers)**.

Архитектура состоит из двух уровней:

1.  **Массив-костяк (Уровень Строк):** Создается одномерный динамический массив, элементами которого являются не сами данные (числа или символы), а _указатели_ (адреса) на другие массивы `(Тип**)` .
2.  **Массивы данных (Уровень Столбцов):** Для каждого элемента массива-костяка (для каждой строки) отдельно запрашивается новый одномерный динамический массив в куче `(Тип*)`. В этих массивах и будут физически храниться полезные данные.

Такая структура логически воспринимается программистом как таблица с двойной индексацией `matrix[i][j]`, но физически в оперативной памяти строки матрицы могут быть "разбросаны" (фрагментированы) по совершенно разным непоследовательным адресам кучи.

---

## Многоступенчатое выделение и освобождение памяти

Сложность архитектуры массива указателей напрямую отражается на алгоритмах его создания и уничтожения. Они становятся итеративными (требуют циклов).

### 1. Этап выделения (Аллокация)

Нельзя выделить двумерный массив одной операцией `new`. Процесс распадается на шаги:

- Сначала выделяется **массив указателей** на строки. Тип переменной, хранящей адрес этого массива, — это _указатель на указатель_ (например, `int**`).
- Затем запускается цикл (от 0 до количества строк). На каждой итерации выделяется одномерный массив (представляющий строку), и его адрес записывается в соответствующую ячейку массива указателей.

### 2. Этап уничтожения (Деаллокация)

Крайне важно понимать: оператор `delete[]` не умеет "догадываться" о сложной вложенной структуре данных. Если удалить только массив-костяк, все массивы-строки навсегда останутся в куче, породив катастрофическую утечку памяти (Memory Leak).
**Удаление должно происходить строго в обратном порядке:**

- Сначала запускается цикл по всем строкам. Для каждого элемента массива-костяка вызывается `delete[]`.
- Только после того, как все строки-массивы уничтожены, вызывается `delete[]` для самого массива-костяка.

## Пример реализации на C++

Классический подход к созданию, заполнению и корректному удалению двумерного динамического массива (матрицы).

```cpp
#include <iostream>
#include <iomanip> // Для форматированного вывода std::setw

void twoDimensionalArrayExample() {
    int rows = 3;
    int cols = 4;

    // --- 1. ВЫДЕЛЕНИЕ ПАМЯТИ ---
    // Шаг 1.1: Создаем "массив указателей" на строки.
    // Тип int** означает: "Указатель, указывающий на указатель(и) типа int".
    int** matrix = new int*[rows];

    // Шаг 1.2: Выделяем память под каждую строку (массив столбцов) отдельно
    for (int i = 0; i < rows; ++i) {
        matrix[i] = new int[cols];
    }

    // --- 2. ЭКСПЛУАТАЦИЯ: Инициализация ---
    // Доступ осуществляется привычным синтаксисом двойной индексации
    int counter = 1;
    for (int i = 0; i < rows; ++i) {
        for (int j = 0; j < cols; ++j) {
            matrix[i][j] = counter++; // Заполняем числами от 1 до 12
        }
    }

    // --- 3. ЭКСПЛУАТАЦИЯ: Вывод на экран ---
    std::cout << "Сгенерированная матрица:\n";
    for (int i = 0; i < rows; ++i) {
        for (int j = 0; j < cols; ++j) {
           std::cout << std::setw(4) << matrix[i][j];
        }
        std::cout << "\n";
    }

    // --- 4. ОСВОБОЖДЕНИЕ ПАМЯТИ (Строго обратный порядок!) ---
    // Шаг 4.1: Сначала уничтожаем все "строки" в цикле
    for (int i = 0; i < rows; ++i) {
        delete[] matrix[i];
    }
    // Шаг 4.2: Уничтожаем сам массив "указателей-строк"
    delete[] matrix;

    // Защита от висячего указателя
    matrix = nullptr;
}

int main() {
    twoDimensionalArrayExample();
    return 0;
}
```

## Протокол прохождения контрольного примера

Контрольный пример: создание матрицы размером `rows = 2` и `cols = 3`.

- **Шаг 1: `int** matrix = new int\*[2];`\*\*
  - _Куча:_ Выделяется один блок под 2 указателя (на 64-битных системах это 2 \* 8 = 16 байт). Предположим, начальный адрес блока `0x1000`.
  - _Стек:_ Создается локальная переменная `matrix` типа `int**` (лежит по адресу на стеке, хранит значение `0x1000`).
- **Шаг 2. Итерация 0: `matrix[0] = new int[3];`**
  - _Куча:_ Выделяется блок под 3 элемента `int` (3 \* 4 = 12 байт). Предположим, ОС нашла свободное место по адресу `0x2000`.
  - _Связывание:_ В ячейку `matrix[0]` (которая по адресу `0x1000`) записывается указатель со значением `0x2000`.
- **Шаг 3. Итерация 1: `matrix[1] = new int[3];`**
  - _Куча:_ Выделяется еще 12 байт. Так как куча работает динамически, этот блок совершенно не обязан располагаться рядом с первым. ОС может выделить его по адресу `0x4080`.
  - _Связывание:_ В ячейку `matrix[1]` (по адресу `0x1008`) записывается значение `0x4080`.
- **Результат:** Мы создали логическую сетку 2x3. Вызов `matrix[1][2]` компилятор транслирует как: "Возьми адрес из `matrix[1]` (`0x4080`), добавь сдвиг `2 * sizeof(int)` и прочитай значение по адресу `0x4088`".
- **Шаг 4. Освобождение (цикл):**
  - Вызов `delete[] matrix[0];` освобождает 12 байт по адресу `0x2000`.
  - Вызов `delete[] matrix[1];` освобождает 12 байт по адресу `0x4080`.
- **Шаг 5. Финальное освобождение:**
  - Вызов `delete[] matrix;` освобождает 16 байт по адресу `0x1000`.

## Архитектурные недостатки (Фрагментация и Кэш)

Реализация "массив массивов" имеет серьезные технические изъяны для высоконагруженных систем:

1.  **Фрагментация памяти (Memory Fragmentation):** Поскольку каждая строка матрицы запрашивается у ОС отдельным вызовом `new`, они могут быть разбросаны далеко друг от друга в физической памяти.
2.  **Промахи кэша (Cache Misses):** Современные процессоры копируют данные из медленной RAM в сверхбыстрый кэш L1/L2 целыми блоками (Cache Lines, обычно по 64 байта). Если физические адреса строк разбросаны, переход от строки `[0]` к `[1]` почти гарантированно приведет к сбросу кэша (Cache Miss), что катастрофически замедляет обработку больших матриц (например, при машинной графике).

**Продвинутая альтернатива (Flattening 2D to 1D):**
В промышленном (production) коде двумерные массивы часто создают в виде _одного сплошного одномерного массива_ размером `rows * cols`.

```cpp
int* flatMatrix = new int[rows * cols];
// Доступ к элементу логической строки 'r' и логического столбца 'c':
// Вычисляем индекс: ряд * количество_колонок + столбец
// flatMatrix[r * cols + c] = 42;
delete[] flatMatrix;
```

Этот подход (Имитация 2D через 1D) требует всего одного `new` и одного `delete[]`, полностью уничтожает фрагментацию и гарантирует идеальное кеширование (Spatial Locality) при последовательном чтении.

## Резюме характеристик

| Характеристика                           | Значение / Оценка                                                  |
| :--------------------------------------- | :----------------------------------------------------------------- |
| **Базовая архитектура**                  | Массив указателей на одномерные массивы                            |
| **Внутренняя структура памяти**          | Фрагментированная (строки неразрывны, но не связаны друг с другом) |
| **Количество аллокаций (`new`)**         | `1 + Количество_Строк`                                             |
| **Количество деаллокаций (`delete`)**    | `1 + Количество_Строк`                                             |
| **Сложность аллокации/удаления**         | `O(N)`, где N - количество строк                                   |
| **Доступ к элементу `[i][j]`**           | `O(1)`, двойное разименование по указателю                         |
| **Пространственная локальность (Сache)** | Низкая (высокий риск cache misses между строками)                  |
