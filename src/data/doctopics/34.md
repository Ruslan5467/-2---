# Способы обхода бинарного дерева. Обратный обход (postOrder)

## Теоретическая база: Обратный обход (Post-order)

Обратный обход (Post-order traversal) также относится к категории алгоритмов "Поиск в глубину" (Depth-First Search, DFS).

Его алгоритмическая формула звучит как **L-R-N** (Left-Right-Node, Лево-Право-Узел).

В отличие от прямого обхода (N-L-R), где родитель обрабатывался первым, при обратном обходе родительский узел (Корень) обрабатывается (посещается) **в самую последнюю очередь**, только после того, как будут полностью и окончательно обработаны все его дети (сначала в левом поддереве, затем в правом).

Правила обхода (строгий порядок):

1. **L (Left):** Рекурсивно спускаемся и обходим всё **Левое поддерево** до самого дна.
2. **R (Right):** Затем прочесываем всё **Правое поддерево** до самого дна.
3. **N (Node):** Только убедившись, что у текущего узла больше нет необработанных потомков, мы "посещаем" (обрабатываем/выводим на экран) сам текущий **Корень/Узел**.

Физически это означает, что обход дерева происходит по принципу "Снизу вверх" (Bottom-Up) от нижних листьев к главному корню. Самым последним элементом, который алгоритм выведет на экран, всегда будет Главный Корень дерева.

## Пример использования (Сборка дерева и обратный обход)

Реализуем функцию `postOrder` на C++, которая выведет элементы в консоль.

```cpp
#include <iostream>

struct TreeNode {
    int data;
    TreeNode* left;
    TreeNode* right;

    TreeNode(int val) : data(val), left(nullptr), right(nullptr) {}
};

// Функция ОБРАТНОГО ОБХОДА (Post-order: Лево -> Право -> Узел)
void postOrder(TreeNode* node) {
    // Базовый случай (дно рекурсии)
    if (node == nullptr) {
        return;
    }

    // L: До талого спускаемся в левое поддерево
    postOrder(node->left);

    // R: Затем спускаемся в правое поддерево
    postOrder(node->right);

    // N: Обработка самого узла (печатаем родителя только после детей)
    std::cout << node->data << " ";
}

// Удаление дерева - это классический пример Post-order обхода!
void deleteTree(TreeNode* node) {
    if (node == nullptr) return;

    deleteTree(node->left);   // Сначала сжигаем мосты слева (L)
    deleteTree(node->right);  // Затем сжигаем мосты справа (R)

    // N: Уничтожаем саму ячейку памяти родителя в куче
    delete node;
}

int main() {
    // Собираем то же самое бинарное дерево:
    //          ( 1 )
    //         /     \
    //      ( 2 )   ( 3 )
    //      /   \       \
    //   ( 4 ) ( 5 )   ( 6 )

    TreeNode* root = new TreeNode(1);
    root->left = new TreeNode(2);
    root->right = new TreeNode(3);

    root->left->left = new TreeNode(4);
    root->left->right = new TreeNode(5);

    root->right->right = new TreeNode(6);

    std::cout << "Результат обратного обхода (Post-order):\n";
    postOrder(root);
    // Ожидаемый вывод: 4 5 2 6 3 1
    std::cout << "\n";

    // Уборка мусора (сборщик тоже пройдется снизу вверх!)
    deleteTree(root);

    return 0;
}
```

## Протокол прохождения (Механика рекурсии `L-R-N`)

Рассмотрим, как формируется вывод `4 -> 5 -> 2 -> 6 -> 3 -> 1` в оперативной памяти (Call Stack):

1. **Корень 1:** Вызываем `postOrder(1)`. Инструкция требует сделать **L**: уйти в левое поддерево. Корень 1 ставится на глубокую паузу (морозится). Он ждет, пока левое поддерево закончит свою работу.
2. **Узел 2:** Инструкция **L**: переходит к левому сыну (4). Узел 2 морозится.
3. **Узел 4:** Инструкция **L**: левого сына нет (`nullptr`), возврат. Инструкция **R**: правого сына нет, возврат. Дошли до инструкции **N**: печатаем `4`! Узел 4 отработал, удаляется из стека и возвращает управление в Узел 2.
4. **Узел 2:** Левая ветвь рапортовала (напечаталась 4). Теперь Узел 2 переходит к инструкции **R**: идет к правому сыну (5).
5. **Узел 5:** Делает шаги **L** и **R** (они пустые), переходит к шагу **N**: печатает `5`. Возвращается в Узел 2.
6. **Полный возврат узла 2:** Теперь у Узла 2 выполнена левая ветка (выведено 4) и правая ветка (выведено 5). Наконец, он имеет право выполнить инструкцию **N** для себя: печатает `2`.
7. **Корень 1 дождался:** Возврат от Узла 2 означает, что шаг **L** для Корня 1 завершен (обработано всё левое поддерево). Корень 1 переходит к шагу **R**: спускается в Узел 3. Узел 1 снова морозится.
8. **Узел 3:** Инструкция **L**: левого сына нет (пусто). Инструкция **R**: переходит в Узел 6.
9. **Узел 6:** Лист. Делает пустые проверки, доходит до инструкции **N** и печатает `6`. Возврат.
10. **Узел 3:** Обе ветви завершены. Инструкция **N**: печатает `3`.
11. **Финал Корня 1:** Возврат от Узла 3. Шаги **L** и **R** завершены! Инструкция **N** Корня 1: печатает `1`. Обход закончен.

Главная визуальная особенность — лист, находящийся на самом глубоком левом "дне", печатается первым. А корневой узел распечатается Строго Последним.

## Практическое применение обратного обхода

Движение снизу-вверх — это фундамент для процедур, которые математически зависят от своих частей:

1. **Безопасное удаление дерева в C++ (Garbage Collection):** Функция `deleteTree` из примера — это эталонный обратный обход. Если вы попытаетесь удалить сначала родителя (`delete node;`), а потом чистить детей (`node->left`), операционная система выдаст ошибку "Access Violation". Вы не можете получить доступ к полю `node->left`, если вы уже уничтожили сам `node`. Поэтому мы _обязаны_ сначала уничтожить детей (снизу) и только потом уничтожать родителя.
2. **Вычисление метрик (например, Высоты или Суммы узлов):** Как мы видели в прошлом вопросе, чтобы родитель мог вычислить свою "высоту", он сначала должен получить данные от своих детей (`calculateHeight`). Родитель ждет ответы от детей (L и R) и только потом делает свой шаг (N: сложение максимумов).
3. **Постфиксная нотация в калькуляторах (Обратная польская запись):** При синтаксическом анализе (AST-парсер) обход математического дерева операций пост-ордером выдаст нам идеальную постфиксную запись, которую процессор (стек) может вычислить в один проход: `A B * C +`. (Где знак ставится после операндов).

## Резюме характеристик

| Характеристика                        | Описание                                                                                                                              |
| :------------------------------------ | :------------------------------------------------------------------------------------------------------------------------------------ |
| **Категория алгоритма**               | DFS (Depth-First Search - Поиск в глубину).                                                                                           |
| **Поведение (Формула)**               | **L-R-N** (Left, Right, Node). Обход "Снизу вверх" (Bottom-Up). Родитель обрабатывается последним.                                    |
| **Асимптотическая сложность (Время)** | $O(N)$, где $N$ — количество узлов в дереве. Мы посещаем каждый узел ровно 1 раз.                                                     |
| **Сложность по памяти**               | $O(H)$, где $H$ — высота дерева (глубина стека вызовов).                                                                              |
| **Применение**                        | Деструктор дерева (`delete`), математические свертки от листьев к корню (вычисление высоты), компиляция AST в Постфиксные инструкции. |
