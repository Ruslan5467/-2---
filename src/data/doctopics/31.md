# Деревья. Основные понятия и определения. Простейший пример определения дерева

## Теоретическая база: Переход к нелинейным структурам данных

Массивы и связные списки, которые мы изучали ранее, относятся к **линейным** структурам данных. В них элементы выстроены строго один за другим, и у каждого элемента есть ровно один предшествующий и ровно один последующий элемент (за исключением первого и последнего). Линейные структуры отлично подходят для хранения очередей, стеков или просто списков однородных объектов.

Но в реальном мире многие системы имеют сложную иерархическую структуру. Например:

- Файловая система на компьютере (Папка -> Подпапки -> Файлы).
- Организационная структура компании (Директор -> Заместители -> Отделы -> Сотрудники).
- Структура HTML-документа (Тег `html` содержит `head` и `body` и так далее).

Для моделирования таких иерархических зависимостей в информатике применяется фундаментальная нелинейная структура данных — **Дерево (Tree)**.

**Дерево** — это абстрактный тип данных, который представляет собой совокупность элементов (узлов), соединенных между собой отношениями "родитель-потомок". Дерево называется так потому, что его графическое представление напоминает перевернутое физическое дерево (корень находится сверху, а ветви растут вниз).

## Фундаментальная терминология графов и деревьев

Для уверенной работы с деревьями программисту необходимо владеть специфическим математическим словарем (взятым из теории графов):

1.  **Узел (Node) / Вершина:** Базовый строительный блок дерева. Клетка памяти, хранящая какие-то полезные данные и указатели (связи) на другие узлы.
2.  **Связь (Edge) / Ребро:** Соединение между двумя узлами, отражающее иерархическую подчиненность. В языке C++ ребро физически представляет собой указатель (Pointer-мост).
3.  **Корень (Root):** Самый первый, главный узел в дереве. Это единственный узел в структуре, который _не имеет родителя_ (в него не ведет ни одна стрелка сверху). От Корня берет начало всё дерево.
4.  **Родитель (Parent):** Узел, находящийся на один уровень выше текущего и имеющий прямую ссылку на него. (Например, папка "Windows" — родитель для папки "System32").
5.  **Потомок / Ребенок (Child):** Узел, находящийся на один уровень ниже текущего. Узел "System32" — потомок для "Windows".
6.  **Братья / Сестры (Siblings):** Узлы, имеющие одного и того же общего родителя.
7.  **Лист (Leaf) / Терминальный узел:** Узел, не имеющий ни одного потомка (детей). В файловой системе Листом является сам текстовый файл.
8.  **Путь (Path):** Уникальная последовательность ссылок (ребер), приводящая от Корня целевому узлу. В настоящем дереве существует _только один замкнутый путь_ от Корня до любой произвольной вершины.
9.  **Поддерево (Subtree):** Любой узел дерева вместе со всеми его прямыми и косвенными потомками можно рассматривать как самостоятельное, полностью независимое дерево. Это свойство делает деревья идеальными кандидатами для рекурсивных методов обработки!
10. **Глубина узла / Уровень (Level):** Количество шагов (ребер), которые необходимо совершить от Корня до данного узла. Корень находится на уровне 0.
11. **Высота дерева (Height):** Максимальная глубина самого дальнего Листа. Равна глубине самой "длинной" ветки в дереве.

## Архитектурные ограничения деревьев

Не любая совокупность узлов и связей является деревом. Чтобы граф официально назывался Деревом в рамках CS (Computer Science), он должен соблюдать два жестких физических ограничения:

- **Однонаправленность связей:** Стрелки иерархии всегда направлены от Родителя к Потомоку. (Потомок, правда, может иметь служебный "обратный" указатель на родителя, но это не формирует цикличности).
- **Правило единого предка:** В любой узел, кроме Корня, должна входить _ровно одна_ стрелка. Узел не может иметь двух или трех прямых родителей.
- **Отсутствие циклов (No loops):** Дерево не должно замыкаться само на себя. Вы не можете стартовать из родителя, спуститься по ветвям вниз и случайно вернуться в этого же родителя. Это нарушает концепцию иерархии и превращает дерево в так называемый Циклический граф (Cyclic Graph).

## Простейший пример определения дерева в C++

Существует множество разновидностей деревьев (N-арные, B-деревья, Красно-черные). Но самой востребованной, фундаментальной структурой в программировании является **Бинарное (Двоичное) дерево (Binary Tree)**.

В бинарном дереве действует правило: _Каждый узел может иметь максимум ДВУХ потомков_ — Левого и Правого. (Может иметь одного, может вообще не иметь, но три и более — запрещено).

Давайте напишем простейшее определение узла бинарного дерева на C++ и соберем руками маленькое "ручное" дерево: Корень и два Листа.

```cpp
#include <iostream>

// 1. ОПРЕДЕЛЕНИЕ СТРУКТУРЫ УЗЛА БИНАРНОГО ДЕРЕВА
struct TreeNode {
    int data;            // Полезная информация (числовой "груз" узла)
    TreeNode* left;      // Указатель на поддерево слева (Левый ребенок/Левая ветвь)
    TreeNode* right;     // Указатель на поддерево справа (Правый ребенок/Правая ветвь)
};

int main() {
    // 2. ВЫДЕЛЕНИЕ ПАМЯТИ В КУЧЕ (АЛЛОКАЦИЯ)
    // Создаем три разрозненных узла. Пока они никак не связаны.
    TreeNode* root = new TreeNode;       // Будущий корень
    TreeNode* leftChild = new TreeNode;  // Будущий лист слева
    TreeNode* rightChild = new TreeNode; // Будущий лист справа

    // 3. ИНИЦИАЛИЗАЦИЯ И СБОРКА ИЕРАРХИИ
    // Настраиваем корень
    root->data = 50;                     // Поместим в корень число 50
    root->left = leftChild;              // Сращиваем мост налево
    root->right = rightChild;            // Сращиваем мост направо

    // Настраиваем левого ребенка (он будет листом)
    leftChild->data = 25;
    leftChild->left = nullptr;           // У листа нет детей (заглушки)
    leftChild->right = nullptr;

    // Настраиваем правого ребенка (он тоже лист)
    rightChild->data = 75;
    rightChild->left = nullptr;
    rightChild->right = nullptr;

    // 4. ТЕСТИРОВАНИЕ: ОБХОД И ПЕЧАТЬ ДЕРЕВА ДО ВТОРОГО УРОВНЯ
    std::cout << "Корень: " << root->data << "\n";

    // Элементарная проверка на nullptr позволяет избежать падения программы
    if (root->left != nullptr) {
        std::cout << " |_ Левый потомок: " << root->left->data << "\n";
    }
    if (root->right != nullptr) {
        std::cout << " |_ Правый потомок: " << root->right->data << "\n";
    }

    // 5. ОЧИСТКА ПАМЯТИ (Для деревьев это сложная рекурсивная задача)
    // В данном ручном примере мы просто удаляем узлы снизу вверх
    delete leftChild;
    delete rightChild;
    delete root;

    return 0;
}
```

## Анализ структуры `TreeNode`

Почему структура узла бинарного дерева именно такая?
Дерево — это развитие системы связного списка. В односвязном списке (Node) у нас был один полезный груз (`data`) и один мост-якорь (`next`). Список был подобен нитке бус — мы могли двигаться только прямо.

Структура бинарного дерева `TreeNode` имеет "разветвитель-тройник". Она содержит тот же `data`, но теперь у нас **два моста**: шанс свернуть на под-ветку `left` или шанс свернуть на под-ветку `right`. Если какой-то из мостов "оборван" (указывает на `nullptr`), это означает, что в этом направлении ветвь не растет.

Благодаря такой двухмерной архитектуре программистам удается строить сверхбыстрые поисковые алгоритмы (например, Бинарные Деревья Поиска - BST), где каждый шаг вниз по дереву отсекает половину ложных вариантов, сокращая время поиска миллионов записей до долей миллисекунды.

## Резюме характеристик

| Термин             | Инженерный смысл                                                                                                                                             |
| :----------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Корень (Root)**  | Единственная точка "захвата" всего древовидного объекта в ОЗУ. Любая потеря адреса `root` превращает всё дерево в недостижимый "мертвый" мусор.              |
| **Лист (Leaf)**    | Узел, у которого все указатели-ветви сброшены (равны `nullptr`). Точка остановки любой рекурсии при проходе вглубь дерева.                                   |
| **Поддерево**      | Свойство фрактальности. Левый указатель Кореня (`root->left`) можно рассматривать как абсолютно законный новый «корень» для изолированного левого поддерева. |
| **Свойство путей** | Чтобы от корня спуститься к нужному узлу, процессор должен проделать серию выборов (Например: Влево -> Вправо -> Влево -> Найдено).                          |

## Контрольные вопросы для самопроверки

- Какая принципиальная алгоритмическая разница (в концепции организации и движения) существует между "линейной" структурой связного списка и "нелинейной" структурой дерева?
- Дайте строгое определение термина "Корень дерева" и термина "Лист дерева" с точки зрения входящих и исходящих из них указателей. Могут ли в дереве существовать два Корня?
- Какое фундаментальное правило графовой маршрутизации будет нарушено, если мы возьмем левого потомка (`leftChild`) и в его поле `right` случайно запишем ссылку на сам `root`? Как это называется в CS и к чему это приведет при обходе структуры алгоритмом?
- Почему в структуре `TreeNode` указатели `left` и `right` объявлены как имеющие тип `TreeNode*` (обращающиеся сами к себе)?
- В примере ручной сборки в функции `main` мы удалили куски памяти вручную (`delete leftChild`, затем `delete root`). Что произойдет с оперативной памятью, если мы сначала вызовем `delete root;`, а про листья забудем? Сможем ли мы удалить их после этого?
