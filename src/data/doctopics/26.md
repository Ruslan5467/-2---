# Структуры в С++. Организация линейного односвязного списка без головного элемента

## Теоретическая база: Динамические структуры данных

До сих пор мы рассматривали хранение данных в массивах (статических или динамических). Главная проблема массива — **непрерывность памяти**. Если мы хотим вставить новый элемент в начало или середину заполненного массива, нам придется физически сдвигать все последующие элементы вправо, что крайне неэффективно (время $O(N)$). Кроме того, увеличение размера массива часто требует выделения нового куска памяти и копирования туда всех старых данных.

Для решения этих проблем существуют **связные списки**.
**Линейный односвязный список** — это базовая динамическая структура данных, состоящая из узлов (элементов), разбросанных в случайных местах оперативной памяти.

Каждый узел не только хранит полезную информацию (данные), но и хранит **указатель** (адрес в памяти) на следующий узел в цепочке. Благодаря этим "ссылкам" узлы программно связываются в единую последовательность, напоминающую вагоны поезда.

### Список без фиктивного головного элемента

В такой архитектуре у нас есть внешний указатель `head` (голова), который сразу указывает на **первый реальный узел** с полезными данными. Если список пуст, указатель `head` равен `nullptr`. У последнего узла в списке указатель на следующий элемент также равен `nullptr`, что служит сигналом окончания цепочки.

---

## Архитектурная модель данных: Узел (Node)

Для создания связанного списка мы используем механизм **Структур (`struct`)**, так как узел должен объединять внутри себя принципиально разные по смыслу типы: сами данные и служебный адрес следующего элемента.

В C++ это реализуется через "самоссылающуюся" структуру. Обратите внимание: структура не может содержать саму себя как объект (это привело бы к бесконечной вложенности и размеру), но она **может** содержать _указатель_ на свой собственный тип!

```cpp
// Объявление узла списка
struct Node {
    int data;       // Полезные данные (допустим, целое число)
    Node* next;     // Указатель на следующий узел в списке (содержит адрес в ОЗУ)
};
```

## Пример использования (Создание, Вставка в начало и Вывод)

Самой быстрой и эффективной операцией для односвязного списка является добавление нового элемента в самое начало (перед текущей "головой"). Это классический паттерн стека (LIFO), когда новые элементы вытесняют старые вглубь.

Напишем программу, организующую список без фиктивного головного узла, добавим в него несколько чисел и последовательно их выведем.

```cpp
#include <iostream>

// 1. ОПРЕДЕЛЕНИЕ СТРУКТУРЫ УЗЛА
struct Node {
    int data;
    Node* next;
};

// Функция добавления нового узла в НАЧАЛО списка
// Обратите внимание: head передается ПО ССЫЛКЕ (Node*&),
// так как мы планируем изменить сам оригинальный указатель head внутри функции!
void pushFront(Node*& head, int value) {
    // 1. Выделяем память в Куче (Heap) для нового узла
    Node* newNode = new Node;

    // 2. Инициализируем данные
    newNode->data = value;

    // 3. Формируем связь: новый узел должен указывать на "старую" голову
    // (даже если старая голова была nullptr - это нормально)
    newNode->next = head;

    // 4. Переносим ярлык "Головы" на наш свежесозданный узел
    head = newNode;
}

// Функция для печати всех элементов списка
// Здесь head передается по значению, так как мы только читаем и не хотим сдвинуть
// оригинальную голову списка в вызывающей функции.
void printList(Node* head) {
    Node* current = head; // Создаем временного "бегунка" для навигации

    std::cout << "Содержимое списка: ";
    // Пока бегунок не спрыгнет в пропасть (не достигнет конца списка)
    while (current != nullptr) {
        std::cout << current->data << " -> ";
        // Шагаем вперед: переходим по адресу, записанному в поле next
        current = current->next;
    }
    std::cout << "NULL\n";
}

// Функция для очистки памяти (Критически важно в C++!)
void clearList(Node*& head) {
    while (head != nullptr) {
        Node* temp = head;       // Запоминаем текущую голову
        head = head->next;       // Сдвигаем голову на следующий узел
        delete temp;             // Уничтожаем старую голову
    }
}

int main() {
    // Инициализация ПУСТОГО списка.
    // Указатель head никуда не ссылается.
    Node* head = nullptr;

    // Добавляем элементы в начало
    std::cout << "Добавляем 10 в начало...\n";
    pushFront(head, 10);
    printList(head); // Ожидаем: 10 -> NULL

    std::cout << "Добавляем 20 в начало...\n";
    pushFront(head, 20);
    printList(head); // Ожидаем: 20 -> 10 -> NULL

    std::cout << "Добавляем 30 в начало...\n";
    pushFront(head, 30);
    printList(head); // Ожидаем: 30 -> 20 -> 10 -> NULL

    // Очистка выделенной памяти
    clearList(head);
    std::cout << "Память очищена.\n";

    return 0;
}
```

## Протокол прохождения: Вставка в начало (`pushFront`)

Разберем хирургическую точность операции добавления числа 30 в список, который уже имеет вид `[20] -> [10] -> NULL`. Оригинальный `head` хранит адрес узла `[20]`.

1.  **Аллокация:** `Node* newNode = new Node;` — операционная система находит где-то в оперативной памяти свободные 12 байт (4 для int, 8 для указателя) и отдает их под новый узел. Пусть адрес нового узла будет `0xABC`.
2.  **Инициализация:** `newNode->data = 30;` — записываем полезную нагрузку.
3.  **Перезапись ссылки (Склейка вагонов):** `newNode->next = head;` — крайне важный шаг. Мы берем адрес текущей головы (адрес узла `[20]`) и аккуратно записываем этот адрес в поле `next` нашего нового узла `[30]`.
    Теперь физически существует цепочка: `[30]` указывает на `[20]`, который указывает на `[10]`.
    _Опасность:_ В этот момент переменная `head` все еще несправедливо указывает на `[20]`.
4.  **Смена короны:** `head = newNode;` — переменной-костылю `head` присваивается новый адрес `0xABC`. Теперь официальное начало списка — это наш новый узел с числом 30.

Вся эта операция — это просто запись нескольких чисел в ячейки памяти. Здесь **нет никакого физического сдвига** соседних элементов (как это было бы в массиве `std::vector::insert(0)`). Именно поэтому вставка в начало списка всегда происходит за идеальное, константное время **$O(1)$**.

## Резюме характеристик

| Характеристика                   | Описание                                                                                                                                                       |
| :------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Парадигма памяти**             | Разрозненные узлы в "Куче" (Heap), связанные указателями-мостами.                                                                                              |
| **Класс без "фиктивной" головы** | Указатель `head` сразу ссылается на первый логический элемент с полезными данными (либо равен `nullptr`, если список пуст).                                    |
| **Доступ к элементу**            | Последовательный (Sequential). Нельзя получить 5-й элемент мгновенно по индексу. Нужно 5 раз прыгнуть по мостам через `current = current->next`. Время $O(N)$. |
| **Вставка в начало**             | Константное время $O(1)$. Не требует сдвига.                                                                                                                   |

## Контрольные вопросы для самопроверки

- В чем заключается фундаментальное архитектурное преимущество связных списков перед традиционными статическими и динамическими массивами в ситуациях, когда требуется часто добавлять новые объекты в самое начало последовательности?
- Почему в структуре `Node` для реализации "цепочки" мы обязаны использовать именно указатель `Node* next`, а не хранить следующий узел обычным вложением `Node next`? Какую ошибку выдаст компилятор С++ при такой попытке?
- Объясните причину, по которой параметр `head` в функции `pushFront(Node*& head, int value)` принимается **по ссылке с указателем** (амперсанд `&`), а в функции печати `printList(Node* head)` он принимается **по значению**? Что сломается в программе, если убрать `&` из `pushFront`?
- Опишите, к каким последствиям (с точки зрения целостности памяти) приведет ситуация, если программист внутри `pushFront` сначала переопределит голову `head = newNode;` и только затем попытается прикрепить старый список к новому узлу `newNode->next = head;`?
- Если нам необходимо узнать общее количество элементов в односвязном списке без головного фиктивного элемента, как мы можем подсчитать это количество? Напишите (ментально или на бумаге) краткий алгоритм итеративного подсчета с помощью локальной переменной-счетчика `count` и цикла `while`.
