# Рекурсивные алгоритмы на примере задачи обхода шахматной доски конем

## Теоретическая база: Перебор с возвратом на графе (Граф состояний)

Задача обхода шахматной доски конем (Knight's Tour) — это еще одна классическая алгоритмическая задача, которая идеально, подобно задаче о ферзях, решается с помощью паттерна **Перебор с возвратом (Backtracking)**.

**Формулировка задачи:** Дана пустая шахматная доска размером $N \times N$. Конь начинает свой путь из заданной начальной клетки. Требуется найти такой маршрут (последовательность ходов) шахматного коня, чтобы он посетил **каждую клетку доски ровно один раз**. Конь ходит по обычным шахматным правилам — буквой «Г» (на две клетки в одном направлении и на одну клетку в перпендикулярном).

С точки зрения информатики, шахматная доска математически представляет собой **Граф (Graph)**, где клетки — это вершины (узлы), а возможные ходы коня из клетки в клетку — это ребра (связи) между вершинами. В таком представлении задача сводится к поиску **Гамильтонова пути** — пути в графе, который проходит через все узлы ровно по одному разу.

Так как аналитической формулы маршрута не существует, единственным базовым способом решения является рекурсивный "слепой" поиск по лабиринту потенциальных ходов с возвратом из тупиков.

---

## Архитектурная модель данных

Для реализации нам потребуется:

1.  **Двумерный массив доски (`board[N][N]`):** В отличие от задачи о ферзях, где нам был важен только факт наличия ферзя в столбце, здесь нам важно отслеживать _сам маршрут_ и _порядковый номер шага_. В каждой клетке мы будем хранить число (от 0 до $N^2 - 1$), означающее номер хода, на котором конь посетил эту клетку. Клетки, в которых конь еще не был, будут инициализированы значением `-1`.
2.  **Массивы возможных смещений (Дельты координат):** Из любой клетки на бесконечной доске конь может сделать максимум 8 вариантов хода. Мы можем закодировать эти смещения (delta X и delta Y) в два константных массива:
    `int dx[8] = { 2, 1, -1, -2, -2, -1, 1, 2 };`
    `int dy[8] = { 1, 2, 2, 1, -1, -2, -2, -1 };`

## Пример использования (Реализация на C++)

Ниже представлена классическая рекурсивная реализация поиска маршрута для нестандартной уменьшенной доски 5x5 (на стандартной 8x8 алгоритм без оптимизации может работать часами).

```cpp
#include <iostream>
#include <vector>
#include <iomanip>

const int N = 5; // Размер доски N x N (для демонстрации используем 5x5)

// Вспомогательная функция: Проверяет, не выходит ли ход за пределы доски
// и свободна ли клетка
bool isSafe(int x, int y, const std::vector<std::vector<int>>& board) {
    return (x >= 0 && x < N && y >= 0 && y < N && board[x][y] == -1);
}

// Печать доски с форматированием (добавление нулей для выравнивания)
void printSolution(const std::vector<std::vector<int>>& board) {
    for (int i = 0; i < N; ++i) {
        for (int j = 0; j < N; ++j) {
            std::cout << std::setw(2) << board[i][j] << " ";
        }
        std::cout << "\n";
    }
}

// Рекурсивный алгоритм Бэктрекинга
// x, y - текущие координаты коня. step - номер хода (от 1 до N*N - 1)
bool solveKT(int x, int y, int step, std::vector<std::vector<int>>& board,
             int dx[], int dy[]) {

    // 1. БАЗОВЫЙ СЛУЧАЙ (Терминальное условие)
    // Если конь сделал N * N шагов (заполнил всю доску), мы победили!
    if (step == N * N) return true;

    // 2. РЕКУРСИВНЫЙ ШАГ
    // Пытаемся сделать ход в одном из 8 доступных направлений (от 0 до 7)
    for (int k = 0; k < 8; ++k) {
        int nextX = x + dx[k];
        int nextY = y + dy[k];

        // Проверяем, законен ли этот ход?
        if (isSafe(nextX, nextY, board)) {
            // ДЕЛАЕМ ШАГ: Конь прыгает в клетку, записываем номер шага
            board[nextX][nextY] = step;

            // РЕКУРСИЯ: Идем глубже, пытаясь распутать маршрут дальше
            // Если вызов вернет true, значит путь из этой ветки ведет к финишу
            if (solveKT(nextX, nextY, step + 1, board, dx, dy)) {
                return true;
            }

            // ВОЗВРАТ (Backtracking): Если путь оказался тупиковым
            // (рекурсия вернула false), отменяем наш ход! Освобождаем клетку.
            board[nextX][nextY] = -1;
        }
    }

    // Если мы перебрали все 8 вариантов и ни один не спас,
    // значит мы загнаны в угол. Возвращаем false.
    return false;
}

int main() {
    std::vector<std::vector<int>> board(N, std::vector<int>(N, -1));

    // Массивы смещения коня
    int dx[8] = { 2, 1, -1, -2, -2, -1,  1,  2 };
    int dy[8] = { 1, 2,  2,  1, -1, -2, -2, -1 };

    // Конь стартует с левого верхнего угла (0,0). Это шаг №0.
    board[0][0] = 0;

    std::cout << "Поиск пути коня для доски " << N << "x" << N << "...\n\n";

    if (solveKT(0, 0, 1, board, dx, dy)) { // Ищем ход под номером 1
        std::cout << "Маршрут успешно найден:\n";
        printSolution(board);
    } else {
        std::cout << "Решение не существует.";
    }

    return 0;
}
```

## Протокол прохождения (Лабиринт Тупиков)

Опишем работу процессора в концепции стека:

1.  **`main()` вызывает `step = 1` для клетки (0,0):**
    - Процессор берет вариант 0 массива: смещение `dx=2, dy=1`.
    - Прыгает в (2,1). Записывает туда число `1`.
    - **Заморозка:** Вызывает `solveKT(..., step = 2)`.
2.  **Погружение (`step = 2 ... 15`):**
    - Алгоритм уверенно шагает по доске дальше, делая успешные ходы, занимая свободные клетки и все глубже погружаясь в рекурсию.
3.  **Тупик (Dead end):** Допустим, на шаге `step = 16` конь прыгнул в угол доски. Окружение заблокировано занятыми клетками или границами доски.
    - Цикл `for (k=0 до 7)` пытается найти свободную клетку.
    - `isSafe` для всех 8 вариантов возвращает `false`.
    - Цикл заканчивается. Функция отвечает: `return false;`. Фрейм `step=16` уничтожается.
4.  **Схлопывание с возвратом (Unwinding / Backtracking):**
    - Процессор "всплывает" обратно в функцию `step = 15`. Она только что получила от своего "клона-потомка" ответ `false`.
    - Срабатывает инструкция возврата: `board[nextX][nextY] = -1;`. Клетка, занятая на 16-м ходу, стирается из памяти, как будто коня там и не было.
    - Алгоритм переходит на следующую итерацию цикла `for (k)` на 15-м уровне и пытается прыгнуть в _другую_ сторону. Если варианты исчерпаны, возвращается `false`, и стирается `step=15`, передавая эстафету 14-му уровню.

## Архитектурные риски: Дерево взрывного роста (Эвристика)

Временная сложность данного наивного алгоритма катастрофична: **$O(8^{N^2})$**.
Король, ищущий путь на доске 8х8, потенциально может создать невероятно гигантское дерево вызовов, перекрывающее количество атомов во вселенной. Перебирать это классическим бэктрекингом — безумие, вычисление может занять тысячи лет.

**Как это решается в инженерии? Эвристика Варнсдорфа.**
Мы не перебираем 8 ходов в случайном (статичном) порядке `dx`/`dy` слева направо. Мы добавляем "интеллект".  
_Правило Варнсдорфа гласит:_ При выборе следующего шага конь должен прыгать в ту клетку, из которой в будущем у него будет **минимальное количество свободных ходов**. Сначала занимаются граничные и угловые клетки-коридоры, оставляя центры масс свободными. Такое простое правило жадного алгоритма (внедренное перед циклом `for (k)`) сокращает время поиска на доске 8x8 с сотен лет до доли миллисекунды.

## Резюме характеристик

| Характеристика                  | Описание                                                                |
| :------------------------------ | :---------------------------------------------------------------------- |
| **Парадигма**                   | Backtracking (Слепой поиск с возвратом из тупиков)                      |
| **Тип графа**                   | Поиск Гамильтонова пути (охват всех вершин ровно один раз)              |
| **Сложность по памяти (Space)** | $O(N^2)$ — глубина стека равна квадрату размера доски                   |
| **Сложность по времени (Time)** | До $O(8^{N^2})$ — в худшем случае (полное древо вариантов без эвристик) |
| **Оптимизация**                 | Жадные алгоритмы на базе эвристик (Правило Варнсдорфа)                  |
