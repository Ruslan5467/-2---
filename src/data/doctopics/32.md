# Бинарные деревья

## Теоретическая база: Суть бинарных деревьев

Деревья в программировании могут иметь любое количество потомков у каждого узла (n-арные деревья). Но наибольшую популярность и практическую значимость приобрел специфический класс деревьев — **Бинарные (Двоичные) деревья**.

**Бинарное дерево** — это иерархическая структура данных, в которой каждый узел имеет **не более двух потомков**. Эти потомки жестко именуются как **"левое поддерево"** (left child) и **"правое поддерево"** (right child).

Даже если у узла есть всего один потомок, нам математически важно знать, _какой именно_ это потомок — левый или правый. Это принципиально отличает бинарные деревья от обычных математических деревьев, где порядок ветвей может не играть роли.

### Архитектурные виды бинарных деревьев:

1.  **Строго бинарное дерево (Strict/Proper Binary Tree):** Дерево, в котором каждый внутренний узел имеет ровно два или ровно ноль дочерних узлов. Узлы ровно с одним ребенком категорически отсутствуют.
2.  **Полное бинарное дерево (Complete Binary Tree):** Дерево, в котором заполнены математически все уровни, кроме, возможно, самого нижнего. При этом нижний уровень заполняется строго слева направо на уровне структуры "листьев". Это математическое свойство используется для реализации структуры "Двоичная Куча" (Heap).
3.  **Идеальное / Совершенное бинарное дерево (Perfect Binary Tree):** Внутренние узлы имеют строго по два потомка, и все листья (терминальные узлы) лежат на одном и том же уровне (глубине). Представляет собой математический идеальный симметричный треугольник.
4.  **Вырожденное дерево (Degenerate Tree):** Дерево, в котором каждый родительский узел имеет только одного ребенка (вытягивается в одну линию). По сути, эта структура алгоритмически деградирует в обычный линейный односвязный список.

## Математические свойства бинарных деревьев

Почему бинарные деревья так любят в Computer Science? Потому что они подчиняются законам степеней двойки, что позволяет сокращать время работы поисковых алгоритмов:

- Максимальное количество узлов на любом уровне `L` равно $2^L$ (где уровень корня — это 0).
  - Уровень 0: $2^0 = 1$ (только корень)
  - Уровень 1: $2^1 = 2$ (левый и правый)
  - Уровень 2: $2^2 = 4$ узла и т.д.
- Максимальное количество узлов во всем Идеальном дереве высотой `H` вычисляется по формуле: $2^{H+1} - 1$.
- В любом непустом строго бинарном дереве, если $N$ — это число листовых узлов, а $K$ — число узлов с двумя детьми, то всегда срабатывает закон: $N = K + 1$.

## Пример использования (Сборка и рекурсивный расчет высоты)

Главной алгоритмической особенностью работы с деревьями является **Рекурсия**. Так как левое и правое поддеревья сами по себе алгоритмически являются абсолютно полноценными структурами "бывших корней", большинство задач (поиск, подсчет, очистка) решаются вызовом функции от самой себя.

Напишем программу, которая собирает небольшое бинарное дерево и вычисляет его максимальную высоту (глубину) рекурсивным способом.

```cpp
#include <iostream>
#include <algorithm> // Для std::max

// 1. ОПРЕДЕЛЕНИЕ СТРУКТУРЫ (Layout памяти)
struct TreeNode {
    int data;            // Значение узла
    TreeNode* left;      // Указатель на поддерево слева
    TreeNode* right;     // Указатель на поддерево справа

    // Для удобства объявим конструктор структуры (С++ позволяет это делать)
    // Это экономит нам место, автоматически зануляя новые ветви.
    TreeNode(int val) {
        data = val;
        left = nullptr;
        right = nullptr;
    }
};

// 2. РЕКУРСИВНАЯ ФУНКЦИЯ ВЫЧИСЛЕНИЯ ВЫСОТЫ
// Возвращает количество уровней в дереве начиная с переданного узла
int calculateHeight(TreeNode* node) {
    // Базовый случай (Terminal condition): дошли до самого дна (пустой ссылки)
    if (node == nullptr) {
        return 0; // Пустое пространство не имеет высоты
    }

    // Рекурсивный спуск:
    // "Делегируем" левому и правому дочерним узлам задачу: "Измерь свою высоту!"
    int leftHeight = calculateHeight(node->left);
    int rightHeight = calculateHeight(node->right);

    // Возвращаемся через стек вызовов наверх:
    // Наша общая высота для этого поддерева — это максимум из веток + 1 (считаем сами себя)
    return std::max(leftHeight, rightHeight) + 1;
}

// 3. РЕКУРСИВНАЯ ОЧИСТКА ПАМЯТИ
void deleteTree(TreeNode* node) {
    if (node == nullptr) return; // Уперлись в тупик — выходим

    deleteTree(node->left);      // Сначала поручаем сжечь всю левую ветку
    deleteTree(node->right);     // Затем сжигаем всю правую ветку
    delete node;                 // Только после обрыва обеих связей уничтожаем тело корня!
}

int main() {
    // Ручная физическая сборка дерева:
    //          (50)         <-- Корень
    //          /  \
    //      (20)    (70)     <-- Уровень 1
    //      /  \      \
    //   (10)  (30)   (90)   <-- Уровень 2
    //           \
    //           (40)        <-- Уровень 3

    // Корень создается в Куче (Heap)
    TreeNode* root = new TreeNode(50);

    // Растим ветви от корня
    root->left = new TreeNode(20);
    root->right = new TreeNode(70);

    // Ветви от левого узла 20
    root->left->left = new TreeNode(10);
    root->left->right = new TreeNode(30);

    // Лист от узла 30
    root->left->right->right = new TreeNode(40);

    // Лист от узла 70
    root->right->right = new TreeNode(90);

    // Выполнение алгоритма (запуск рекурсии)
    int height = calculateHeight(root);

    std::cout << "Максимальная высота дерева равна: " << height << " уровням(я).\n";
    // Корректный ответ: 4 (поскольку узлы лежат на 4-х разных уровнях, включая корень).

    // Запускаем рекурсивный сборщик мусора
    deleteTree(root);

    return 0;
}
```

## Протокол прохождения (Движение по Call Stack)

Разберем, как алгоритм `calculateHeight` обрабатывает левую часть дерева, стартуя от корня `root(50)`.

1.  **Точка входа:** Корень `50` спрашивает свою левую ветвь `calculateHeight(20)`. Программа "замораживает" узел `50` в системном стеке памяти.
2.  **Спуск:** В узле `20` та же функция просит левую ветвь — узел `10`. Узел `20` "заморожен". Узел `10` просит своего левого несуществующего потомока `calculateHeight(nullptr)`.
3.  **Отражение ото дна:** Функция видит базовый случай `if (node == nullptr) return 0;` и возвращает $0$ обратно в вызвавший её узел `10`.
4.  **Сложение:** То же отражение происходит для правого `nullptr` узла 10. `max(0, 0) + 1 = 1`. Узел `10` отчитался! Он возвращает высоту `1` вверх к своему родителю (в узел `20`).
5.  **Параллельная ветвь:** Для вычисления правой ветви узел `20` запрашивает узел `30`. Тот в свою очередь спускается до листа `40`. Лист `40` возвращает высоту `1` вверх.
6.  **Узел 30:** Получает от своего пустого левого "детеныша" ответ `0`, а от правого (`40`) — `1`. Он делает вычисление: `max(0, 1) + 1 = 2`. Узел `30` отчитывается наверх (к узлу `20`), что высота его стороны равна $2$.
7.  **Узел 20:** Наконец-то дожидается всех ответов! Слева пришла высота `1` (со стороны `10`), справа пришла высота `2` (со стороны `30`). Он вычисляет `max(1, 2) + 1 = 3`.
8.  **Финал в Корне:** Значение `3` долетает Корню `50`. Параллельная аналогичная "волна" собирает правое дерево и возвращает Корню значение `2` (так как по правой стороне листьев меньше).
9.  **Итог:** Корень `50` делает финальный аккорд `max(3, 2) + 1 = 4`. Высота вычислена!

С точки зрения архитектуры компьютера, мы использовали **Стек вызовов (Call Stack)**, чтобы процессор мог "запоминать" обратный путь по веткам вверх.

## Резюме характеристик

| Характеристика                      | Описание                                                                                                                                                                                   |
| :---------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Топология**                       | Максимум два потомка (строго именованных: Левый и Правый).                                                                                                                                 |
| **Основной алгоритмический подход** | Использование Рекурсии (мощь дерева в том, что оно фрактально состоит из меньших таких же деревьев).                                                                                       |
| **Математическое свойство**         | Количество узлов на уровне растет по степеням корня из 2. (1, 2, 4, 8, 16...).                                                                                                             |
| **Главный деградационный риск**     | Вырождение бинарного дерева в односвязный список, когда элементы всегда добавляются исключительно в одну сторону. Потеря преимуществ разделения поиска (время ломается в линейное $O(N)$). |

## Контрольные вопросы для самопроверки

- Чем бинарное дерево концептуально отличается от обычного "n-арного" дерева? Почему узлы, имеющие только одного ребенка, имеют фундаментальное значение в контексте "левый/правый" (т.е. левая ветвь не то же самое, что правая ветвь)?
- Объясните значение формулы $2^L$. Точно вычислите, сколько "листьев" или "ячеек" теоретически максимально может физически поместиться на четвертом уровне дерева (где уровень корня — это уровень 0)?
- В примере программы в структуре узла `TreeNode` мы использовали так называемый "Конструктор". В чем его практическое преимущество перед процедурой ручного выделения памяти и построкового зануления указателей `left = nullptr; right = nullptr;` прямо в функции `main()`?
- Разберите рекурсивную функцию `deleteTree(TreeNode* node)`. Почему критически важно вызывать оператор `delete node;` строго ПОСЛЕ рекурсивных вызовов левой и правой ветки (обход bottom-up), а не ДО них? Что бы произошло в памяти, если бы вы удалили тело корня первым?
- Дайте определение "Вырожденному дереву" (Degenerate Tree). Представьте ситуацию: программист в пустое бинарное дерево по очереди, по логике "добавляем всегда только как левого ребенка к последнему листу", добавил числа `1`, `2`, `3`, `4`. Какой вид примет данная физическая структура в оперативной памяти?
