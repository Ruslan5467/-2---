# Способы обхода бинарного дерева. Симметричный обход (inOrder)

## Теоретическая база: Симметричный обход (In-order)

Симметричный обход (In-order traversal) — это третий классический алгоритм из семейства "Поиск в глубину" (Depth-First Search, DFS).

Его алгоритмическая формула звучит как **L-N-R** (Left-Node-Right, Лево-Узел-Право).

Название "симметричный" (или "внутренний") происходит от того, что родительский узел (Корень) обрабатывается (посещается) **строго между** обработкой своего левого поддерева и правого поддерева.

Правила обхода (строгий порядок):

1. **L (Left):** Сначала рекурсивно и глубоко обходим всё **Левое поддерево**.
2. **N (Node):** Затем "посещаем" (обрабатываем/выводим на экран) сам текущий **Корень/Узел**.
3. **R (Right):** И только после этого рекурсивно обходим всё **Правое поддерево**.

Математическая и алгоритмическая красота этого метода заключается в том, что если применить его к специальной структуре "Бинарное Дерево Поиска" (BST, где слева числа меньше родителя, а справа — больше), то симметричный обход выведет все элементы дерева **строго в отсортированном виде (по возрастанию)**.

## Пример использования (Сборка дерева и симметричный обход)

Напишем функцию `inOrder` на C++, которая выведет элементы в консоль. Для наглядности мы соберем дерево, подчиняющееся правилу Бинарного Дерева Поиска.

```cpp
#include <iostream>

struct TreeNode {
    int data;
    TreeNode* left;
    TreeNode* right;

    TreeNode(int val) : data(val), left(nullptr), right(nullptr) {}
};

// Функция СИММЕТРИЧНОГО ОБХОДА (In-order: Лево -> Узел -> Право)
void inOrder(TreeNode* node) {
    // Базовый случай (дно рекурсии)
    if (node == nullptr) {
        return;
    }

    // L: Сначала до талого спускаемся в левое поддерево
    inOrder(node->left);

    // N: Затем обрабатываем сам узел (печатаем между левым и правым)
    std::cout << node->data << " ";

    // R: И наконец спускаемся в правое поддерево
    inOrder(node->right);
}

// Удаление дерева
void deleteTree(TreeNode* node) {
    if (node == nullptr) return;
    deleteTree(node->left);
    deleteTree(node->right);
    delete node;
}

int main() {
    // Собираем Бинарное Дерево Поиска (BST):
    // Узлы слева всегда меньше родителя, справа — больше.
    //          ( 50 )
    //         /      \
    //      ( 25 )   ( 75 )
    //      /   \       \
    //   ( 10 ) ( 30 )   ( 90 )

    TreeNode* root = new TreeNode(50);
    root->left = new TreeNode(25);
    root->right = new TreeNode(75);

    root->left->left = new TreeNode(10);
    root->left->right = new TreeNode(30);

    root->right->right = new TreeNode(90);

    std::cout << "Результат симметричного обхода (In-order):\n";
    inOrder(root);
    // Ожидаемый вывод: 10 25 30 50 75 90
    std::cout << "\n";

    deleteTree(root);

    return 0;
}
```

## Протокол прохождения (Механика рекурсии `L-N-R`)

Рассмотрим, как формируется феномен сортированного вывода `10 -> 25 -> 30 -> 50 -> 75 -> 90` в оперативной памяти (Call Stack):

1. **Корень 50:** Вызов `inOrder(50)`. Инструкция **L**: Корень морозится в стеке (он еще не напечатан!) и запускает обход левого поддерева (переход к узлу 25).
2. **Узел 25:** Инструкция **L**: Узел 25 морозится (он тоже еще не напечатан!) и делает прыжок влево (переход к узлу 10).
3. **Узел 10:** Инструкция **L**: Прыжок влево (в `nullptr`). Возврат (дно).
   - Инструкция **N**: Печатаем Узел `10`! Это первое число, появившееся на экране.
   - Инструкция **R**: Прыжок вправо (в `nullptr`). Возврат.
     Узел 10 отработал и снимается со стека.
4. **Возврат в Узел 25:** Для Узла 25 завершился левый шаг **L** (обработан 10). Теперь наступила его очередь!
   - Инструкция **N**: Печатаем Узел `25`!
   - Инструкция **R**: Прыжок вправо в Узел 30.
5. **Узел 30:** Инструкция **L**: пусто. Инструкция **N**: печатаем `30`! Инструкция **R**: пусто. Возврат из Узла 30 к родителю Узелу 25.
6. **Полный возврат узла 25:** Узел 25 завершил все свои шаги L-N-R. Он возвращает управление в Главный Корень (50).
7. **Корень 50 дождался:** Левое поддерево (содержащее 10, 25, 30) полностью распечатано. Настала очередь Главного Корня выполнить свой шаг.
   - Инструкция **N**: Печатаем Узел `50`!
   - Инструкция **R**: Прыжок в правое поддерево к Узлу 75.
8. **Узел 75:** Инструкция **L**: пусто (левой ветки нет). Инструкция **N**: печатаем `75`! Инструкция **R**: прыжок в Узел 90.
9. **Узел 90:** Шаг L пуст. Печатаем `90`! Шаг R пуст.
10. **Финал:** Все возвраты каскадно схлопываются (90 -> 75 -> 50). Программа штатно завершается.

Главная визуальная особенность — вывод начинается с самого "глубокого" левого элемента и плавно, змейкой переходит направо, проходя через предков.

## Практическое применение симметричного обхода

Существует всего несколько структур данных, обход которых тесно связан именно с L-N-R:

1. **Сортировка Бинарного Дерева Поиска (BST):** Это самое главное производственное назначение симметричного обхода. Если данные хранятся в BST, `inOrder` алгоритм за один линейный проход $O(N)$ «разворачивает» и выхватывает из дерева все элементы в математически отсортированном порядке (от меньшего к большему). Это часто используется для создания так называемого "плоского отсортированного массива" (flattening tree).
2. **Сортировка дерева по убыванию (Reverse In-order):** Если искусственно перевернуть алгоритм в формулу **R-N-L** (сначала Право, потом Корень, потом Лево), то алгоритм выведет элементы BST строго от большего к меньшему, что идеально подходит для создания таблиц лидеров (Leaderboards).
3. **Генерация инфиксной записи выражений:** Если бинарное дерево представляет собой математическое выражение — Синтаксическое дерево (AST) — то симметричный обход выдаст выражение в классическом, понятном человеку виде (Инфиксная нотация): `A + B * C`. Однако в этом случае в алгоритм придется запрограммировать дополнительный вывод скобок `(`, `)`, чтобы сохранить инженерный порядок арифметических операций.

## Резюме характеристик

| Характеристика                        | Описание                                                                                                                                                                        |
| :------------------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Категория алгоритма**               | DFS (Depth-First Search - Поиск в глубину).                                                                                                                                     |
| **Поведение (Формула)**               | **L-N-R** (Left-Node-Right) или **R-N-L** (Права-Узел-Лево). Узел обрабатывается "между" детьми.                                                                                |
| **Асимптотическая сложность (Время)** | $O(N)$, где $N$ — количество узлов в дереве. Мы посещаем каждый узел ровно 1 раз.                                                                                               |
| **Применение**                        | Вывод значений деревьев BST в отсортированном виде (Flatten Array), генерация инфиксной нотации AST (человекочитаемые формулы), поиск k-го минимального/максимального элемента. |

## Контрольные вопросы для самопроверки

- Как расшифровывается алгоритмический паттерн L-N-R в контексте симметричного обхода, и почему он называется "симметричным"?
- Объясните математическую причину, по которой симметричный обход (inOrder) Бинарного Дерева Поиска (BST) гарантированно выводит узлы в отсортированном виде по возрастанию.
- Какую модификацию нужно внести в код симметричного обхода (поменять шаги `inOrder`), чтобы заставить BST дерево выводить свои значения-числа в порядке убывания (от самого большого к самому маленькому)? Как выглядит формула такого обхода?
- В примере (корень $50$, левый ребенок $25$, правый-правый внук левого узла $30$). Узел $25$ был напечатан раньше корня $50$. А почему правый-правый лист $30$ все равно вывелся раньше, чем главный корень $50$?
- Сравните Прямой (N-L-R), Обратный (L-R-N) и Симметричный (L-N-R) обходы. Какой из трех обходов категорически необходимо применять в деструкторах класса (когда нужно очистить динамическую память дерева `delete`), и почему два других алгоритма уничтожат программу (Access Violation)?
