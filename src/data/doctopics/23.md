# Структуры в С++. Пример использования массива структур при решения задачи сортировки

## Теоретическая база: Пользовательские типы данных (Структуры)

Все базовые типы данных в C++ (такие как `int`, `double`, `char`) способны хранить только одно атомарное значение. Однако в реальном мире объекты описываются совокупностью различных характеристик. Например, "Студент" — это не просто возраст (`int`), это еще и имя (`std::string`), и средний балл (`double`).

Для объединения логически связанных данных разного типа под одним именем в языке C++ используется механизм **Структур (Structures, `struct`)**. Структура позволяет программисту создать свой собственный, композитный (пользовательский) тип данных.

В отличие от массивов, где все элементы обязаны быть одного типа, элементы внутри структуры (называемые **полями** или **членами** структуры) могут иметь абсолютно любой тип.

**Объявление структуры:**

```cpp
// Объявление нового типа данных "Студент"
struct Student {
    std::string name; // Имя (строка)
    int age;          // Возраст (целое число)
    double gpa;       // Средний балл (вещественное число)
};
// ВАЖНО: После закрывающей скобки структуры обязательно ставится точка с запятой!
```

## Архитектура памяти при работе со структурами

Когда мы просто объявляем `struct Student { ... };`, компилятор не выделяет ни одного байта оперативной памяти. Мы лишь создали "чертеж" для будущего объекта.
Память выделяется только в момент объявления конкретной переменной (экземпляра) этого типа:
`Student s1;`

Поля структуры располагаются в оперативной памяти **последовательно**, одно за другим (возможно, с добавлением выравнивающих пустых байтов — padding, для оптимизации работы процессора).
Доступ к конкретному полю структуры осуществляется через **оператор доступа (точку `.` )**:

```cpp
s1.name = "Иван";
std::cout << s1.age;
```

## Массивы структур и задача Сортировки

Возможность создавать массивы из пользовательских структур — мощнейший архитектурный инструмент. Вместо создания трех разных массивов (`string names[]`, `int ages[]`, `double gpas[]`), которые сложно синхронизировать при сортировке, мы создаем один массив объектов `Student[]`.

При сортировке массива структур мы должны осуществлять сравнение по одному (опорному) полю, а перемещать (менять местами) **весь объект структуры целиком**.

### Пример использования (Сортировка массива структур)

Реализуем классическую сортировку "Пузырьком" (Bubble Sort) массива из трех студентов. В качестве опорного ключа (критерия) сортировки выберем средний балл (`gpa`), и отсортируем список по убыванию (от отличников к двоечникам).

```cpp
#include <iostream>
#include <string>
#include <vector>

// 1. ОПРЕДЕЛЕНИЕ ТИПА ДАННЫХ
struct Student {
    std::string name;
    int groupNumber;
    double gpa;
};

// Функция для печати таблицы студентов
void printStudents(const std::vector<Student>& arr) {
    std::cout << "-----------------------------------\n";
    std::cout << " Имя\t\tГруппа\tБалл\n";
    std::cout << "-----------------------------------\n";
    for (const auto& student : arr) {
        std::cout << " " << student.name << "\t\t"
                  << student.groupNumber << "\t"
                  << student.gpa << "\n";
    }
    std::cout << "-----------------------------------\n\n";
}

int main() {
    // 2. ИНИЦИАЛИЗАЦИЯ МАССИВА СТРУКТУР
    std::vector<Student> students = {
        {"Борис", 101, 3.8},
        {"Анна",  102, 4.9},
        {"Виктор", 101, 4.2}
    };

    std::cout << "Исходный массив (До сортировки):\n";
    printStudents(students);

    // 3. АЛГОРИТМ СОРТИРОВКИ (По убыванию GPA)
    int n = students.size();
    for (int i = 0; i < n - 1; ++i) {
        for (int j = 0; j < n - i - 1; ++j) {

            // СРАВНЕНИЕ: Сравниваем только одно опорное поле (gpa)
            if (students[j].gpa < students[j + 1].gpa) {

                // ОБМЕН (SWAP): Мы не меняем баллы местами!
                // Мы меняем местами структуры (объекты) ЦЕЛИКОМ!
                // Все поля (Имя, Группа, Балл) автоматически переезжают вместе с объектом.
                Student temp = students[j];
                students[j] = students[j + 1];
                students[j + 1] = temp;

                // Примечание: В реальном коде лучше использовать std::swap(students[j], students[j+1]);
            }
        }
    }

    std::cout << "Отсортированный массив (По убыванию балла):\n";
    printStudents(students);

    return 0;
}
```

## Протокол прохождения контрольного примера

Проследим первый проход (итерацию) внутреннего цикла `j` алгоритма пузырьковой сортировки:

**Исходный массив:**
`[0]: { Борис, 3.8 }`
`[1]: { Анна, 4.9 }`
`[2]: { Виктор, 4.2 }`

- **Шаг 1: `j=0`.** Сравниваем элементы с индексами 0 и 1: `students[0].gpa` (3.8) < `students[1].gpa` (4.9)?
  - **Да (3.8 < 4.9).** Условие убывания нарушено, Борис должен стоять ниже Анны.
  - **Обмен:** Во временную переменную `temp` копируется **ВЕСЬ** профиль Бориса. Слот `[0]` заполняется профилем Анны. В слот `[1]` записывается Борис из `temp`.
  - _Промежуточный массив:_ `[{Анна, 4.9}, {Борис, 3.8}, {Виктор, 4.2}]`.

- **Шаг 2: `j=1`.** Сравниваем элементы с индексами 1 и 2: `students[1].gpa` (3.8) < `students[2].gpa` (4.2)?
  - **Да (3.8 < 4.2).** Виктор учится лучше Бориса, он должен всплыть.
  - **Обмен:** Весь объект "Борис" уходит в `temp`. В слот `[1]` записывается "Виктор". В слот `[2]` опускается "Борис".
  - _Промежуточный массив:_ `[{Анна, 4.9}, {Виктор, 4.2}, {Борис, 3.8}]`.

Первая итерация внешнего цикла завершена. Элемент с самым низким приоритетом (Борис, 3.8) опустился на самое "дно" массива (в конец). Массив по счастливой случайности уже отсортирован полностью, но внешние циклы сделают еще один проход для проверки.

## Продвинутая сортировка с использованием std::sort

В индустриальном коде никогда не пишут "пузырек" вручную. Разработчики используют встроенную функцию `std::sort` и передают в нее **Компаратор (функцию сравнения)**. Это позволяет в один вызов отсортировать массив структур по любому полю:

```cpp
#include <algorithm>

// Компаратор (Правило сравнения): Возвращает логическую Истину, если a должно стоять ПЕРЕД b.
bool compareByName(const Student& a, const Student& b) {
    return a.name < b.name; // Сортировка по алфавиту имени (по возрастанию)
}

// Вызов в main:
// std::sort(students.begin(), students.end(), compareByName);
```

## Резюме характеристик

| Характеристика                 | Описание                                                                                                                                       |
| :----------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------- |
| **Суть структуры (`struct`)**  | Механизм объединения разнородных данных в один монолитный пользовательский тип.                                                                |
| **Массивы структур**           | Предотвращают "рассинхронизацию" характеристик объекта (в отличие от использования параллельных независимых массивов).                         |
| **Синтаксическая особенность** | Обязательное наличие `;` (точки с запятой) после закрывающей скобки объявления структуры. Доступ к полям только через оператор `.`.            |
| **Механика сортировки**        | Сравнение в условии `if` происходит только по ключевому полю, а физический обмен значениями (swap) выполняется для целой структуры-контейнера. |
