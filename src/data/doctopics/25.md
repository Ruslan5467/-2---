# Структуры в С++. Чтение структуры из бинарного файла

## Теоретическая база: Обратный процесс (Десериализация)

Как мы обсуждали ранее, бинарный файл представляет собой сплошной поток сырых байтов, являющийся "слепком" оперативной памяти программы. Чтение из такого файла должно производиться строго по тем же правилам и типам данных, по которым этот файл создавался. Процесс восстановления структур данных из бинарного формата обратно в оперативную память часто называют **десериализацией**.

Для чтения бинарных файлов в C++ используется поток файлового ввода `std::ifstream` с обязательным включением флага `std::ios::binary`.
Чтение массива байтов осуществляется встроенным методом потока `.read()`, который является зеркальным аналогом метода записи `.write()`.

**Важные архитектурные принципы безопасности:**

1.  **Строгое совпадение POD-типов:** При чтении обязательно должна использоваться структура с точно таким же бинарным интерфейсом (layout'ом), какая использовалась при записи. Количество полей, их типы и порядок должны быть идентичны, иначе байты "съедут", и вы получите искаженные данные (мусор).
2.  **Размер файла:** Заранее размер массива структур в файле может быть неизвестен. В таком случае программистам приходится узнавать размер файла в байтах, чтобы разделить его на размер одной структуры (найдя количество элементов) или использовать чтение порциями до символа EOF (Конец файла - End Of File).

## Пример использования (Чтение массива структур из файла)

Напишем программу, которая открывает бинарный файл `database.bin` (созданный в предыдущем вопросе) и считывает дамп базы магазина товаров обратно в массив структур, после чего выводит их на экран.

```cpp
#include <iostream>
#include <fstream>
#include <vector>

// 1. ОПРЕДЕЛЕНИЕ POD-СТРУКТУРЫ
// Должна 100% совпадать со структурой, использованной при записи.
struct Product {
    int id;               // Артикул
    char name[32];        // Статический массив для имени
    double price;         // Цена
};

int main() {
    const char* filename = "database.bin";

    // 2. ОТКРЫТИЕ ФАЙЛА ДЛЯ ЧТЕНИЯ В БИНАРНОМ РЕЖИМЕ
    // ios::in - для чтения, ios::binary - сырые байты
    std::ifstream inFile(filename, std::ios::in | std::ios::binary);

    // Проверка, существует ли файл и доступен ли он?
    if (!inFile.is_open()) {
        std::cerr << "Критическая ошибка: Файл " << filename << " не найден!\n";
        return 1;
    }

    // 3. ОПРЕДЕЛЕНИЕ РАЗМЕРА ФАЙЛА (Чтобы узнать, сколько товаров внутри)

    // Перемещаем "головку чтения" в самый конец файла (Конец = End)
    inFile.seekg(0, std::ios::end);
    // Узнаем, на каком байте остановилась головка (Это и есть размер файла)
    size_t fileSize = inFile.tellg();
    // Обязательно возвращаем головку обратно в начало для последующего чтения!
    inFile.seekg(0, std::ios::beg);

    // Вычисляем размер одного товара
    size_t productSize = sizeof(Product);

    // Вычисляем количество товаров в файле (Делим общий размер на размер штуки)
    size_t productCount = fileSize / productSize;

    // 4. ВЫДЕЛЕНИЕ ПАМЯТИ ПОД ДАННЫЕ
    // Создаем динамический массив нужного размера для загрузки из файла
    std::vector<Product> catalog(productCount);

    // 5. БИНАРНОЕ ЧТЕНИЕ (Главный шаг)
    // Подсказка компилятору: "Возьми адрес начала вектора, забудь про его тип
    // и залей туда fileSize сырых байтов прямо с жесткого диска"
    inFile.read(reinterpret_cast<char*>(catalog.data()), fileSize);

    // 6. ЗАКРЫТИЕ ФАЙЛА
    inFile.close();

    // 7. ВЫВОД РЕЗУЛЬТАТА ДЛЯ ПРОВЕРКИ
    std::cout << "Успешно загружено " << productCount << " товаров из файла:\n";
    std::cout << "-------------------------------------------\n";
    for (size_t i = 0; i < catalog.size(); ++i) {
        std::cout << "ID: " << catalog[i].id
                  << " | Наименование: " << catalog[i].name
                  << " | Цена: $" << catalog[i].price << "\n";
    }
    std::cout << "-------------------------------------------\n";

    return 0;
}
```

## Протокол прохождения (Реверсирование дампа)

Разберем механику чтения:

1.  **Навигация (Seek).** Чтобы прочитать массив неизвестной длины, процессор должен сначала понять размер буфера, который необходимо зарезервировать в ОЗУ. Метод `.seekg(0, std::ios::end)` прыгает мимо всех байтов в самый финал файла. Метод `.tellg()` возвращает номер позиции (по сути — размер в байтах, например `132`).
2.  **Математика блоков.** Процессор вычисляет, что если весь файл "весит" `132` байта, а размер `sizeof(Product)` возвращает `44` байта, значит, в файле хранится ровно `132 / 44 = 3` товара. Вектор растягивается до размера 3.
3.  **Восстановление ОЗУ (Read).** Вызов метода `.read(reinterpret_cast<char*>(...), 132);` инициирует аппаратный I/O процесс. ОС дает команду контроллеру диска физически считать `132` байта начиная с сектора, где лежит `database.bin`. Эти байты без всякого парсинга выгружаются поверх выделенной области памяти (`catalog.data()`).
4.  **Сложение типизирующих рамок.** В ту секунду, когда метод `read` заканчивает работу, массив `catalog` "оживает". Компилятор C++ больше не считает эту область "сырыми байтами". При обращении к `catalog[0].price` процессор знает (благодаря структуре): нужно пропустить в этом блоке первые 36 байт (это 4-байтный `int` и 32-байтный `char`), взять следующие 8 байт и интерпретировать их по стандарту IEEE-754 как `double`!

## Резюме характеристик

| Характеристика             | Описание / Синтаксис                                                                                                                                           |
| :------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------- |
| **Библиотека и класс**     | Заголовочный файл `<fstream>`, класс потока ввода `std::ifstream`                                                                                              |
| **Флаг бинарного режима**  | `std::ios::in                                                                                                                                                  | std::ios::binary` |
| **Ключевой метод чтения**  | `inFile.read(адрес_в_памяти, количество_байт);`                                                                                                                |
| **Определение размера**    | С использованием методов `.seekg()` (для прыжков) и `.tellg()` (для выяснения текущей байт-позиции внутри файла).                                              |
| **Приведение типа адреса** | Принудительный каст: `reinterpret_cast<char*>(&кусок_памяти)` (Заметьте: тут `char*` без спецификатора `const`, так как мы будем изменять / заполнять память). |

## Контрольные вопросы для самопроверки

- В чем заключается основное правило для структуры данных C++ при чтении бинарного файла, если этот файл был записан на другом компьютере или другой программой? Какие два "компонента" макета структуры обязаны совпадать?
- Какова механика работы пары методов `seekg` и `tellg`? Объясните пошагово, каким образом с их помощью язык C++ позволяет узнать точный размер загружаемого бинарного файла в байтах.
- Зачем после измерения размера файла с помощью перевода головки чтения в позицию `std::ios::end` в примере была выполнена команда `inFile.seekg(0, std::ios::beg);`? Что произошло бы при попытке вызова `.read()` без этой промежуточной команды?
- Почему в методе `write()` мы использовали оператор `reinterpret_cast<const char*>`, а в методе `.read()` в операторе отсутствует слово `const` — `reinterpret_cast<char*>`?
- Что произойдет (на физическом уровне оперативной памяти), если вы попытаетесь с помощью `.read()` залить 100 байт из бинарного файла в массив, для которого команда `std::vector` выделила в ОЗУ всего 50 байт физической емкости (емкость меньше размера файла)?
