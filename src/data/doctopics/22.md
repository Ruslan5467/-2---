# Полный перебор в k-ричной системе счисления на примере задачи о ферзях

## Теоретическая база: Изоморфизм перебора массивов и k-ричного счета

Ранее задача о расстановке N ферзей на доске $N \times N$ была решена с помощью интеллектуального рекурсивного перебора (Backtracking). Тот алгоритм был эффективным благодаря раннему отсечению неверных ветвей. Однако в информатике существует и другой, парадигмально иной подход к перебору комбинаций — **Итеративная генерация в k-ричной системе счисления**.

Представим задачу: нам нужно перебрать все возможные комбинации элементов одномерного массива длины $M$, где каждый элемент может принимать значения от $0$ до $k-1$.

Если мы присмотримся к этой задаче, то увидим, что эта математическая модель **абсолютно идентична обычному счету** в позиционной системе счисления с основанием $k$ ($k$-ричной системе).
Например, если нам нужен массив из 3 элементов со значениями от 0 до 9, то это не что иное, как перебор всех чисел от `000` до `999` в нашей привычной десятичной системе.
Если элементы могут быть только от 0 до 1 (битовая маска), то это перебор двоичных чисел от `000...0` до `111...1`.

**Суть алгоритма (Имитация механического одометра):**
Алгоритм представляет собой счетчик, работающий по принципу барабана спидометра в классическом автомобиле:

1. Пытаемся увеличить самый младший разряд (правый элемент массива) на 1.
2. Если он достигает предела (упирается в цифру $k-1$), мы сбрасываем его в $0$ и переносим единицу (эффект carry) в следующий, более старший разряд (сдвигаемся влево).
3. Процесс повторяется, пока самый старший разряд не переполнится.

Этот подход является **чистым полным перебором (Brute-force) без отсечений**.

## Применение математической модели к задаче N-Queens

В задаче о ферзях мы решили хранить расстановку в одномерном массиве длины $N$, где индекс элемента `col` — это номер столбца, а значение `board[col]` — номер строки (от $0$ до $N-1$), в которой стоит ферзь.

Очевидно, что для гарантии перебора _всех без исключения_ позиций нам нужно сгенерировать все возможные массивы из $N$ элементов, где каждый элемент принимает значения от $0$ до $N-1$.
Следовательно, основание нашей системы счисления равно $k = N$.
Нам нужно последовательно прибавлять единицу к длинному числу в $N$-ичной системе счисления, шаг за шагом получая каждую из $N^N$ возможных комбинаций ферзей, и "в лоб" проверять каждую из них на соответствие правилам шахмат.

## Пример использования (Итеративная реализация на C++)

```cpp
#include <iostream>
#include <vector>
#include <cmath>

// Функция проверки ПОЛНОСТЬЮ расставленной доски.
// В отличие от бэктрекинга, мы не можем проверять доску частично по мере роста,
// так как алгоритм одометра всегда выдает завершенный вектор длины N.
bool isValidBoard(const std::vector<int>& queens, int N) {
    for (int col = 0; col < N; ++col) {
        for (int prevCol = 0; prevCol < col; ++prevCol) {
            // Проверяем горизонталь и диагонали (вертикаль гарантируется массивом)
            if (queens[col] == queens[prevCol] ||
                std::abs(queens[col] - queens[prevCol]) == col - prevCol) {
                return false;
            }
        }
    }
    return true;
}

int main() {
    int N = 8; // Размер доски и основание системы счисления (k = 8)
    std::vector<int> queens(N, 0); // Массив, представляющий число из N старших разрядов
    int solutionsCount = 0;
    long long totalCombinations = 0;

    // Бесконечный цикл генерации (аналог while(true))
    while (true) {
        totalCombinations++;

        // 1. ИЗОЛИРОВАННАЯ ПРОВЕРКА сгенерированной комбинации
        if (isValidBoard(queens, N)) {
            solutionsCount++;
        }

        // 2. ИТЕРАТИВНЫЙ ШАГ: Механика "Прибавления +1" в N-ичной системе
        int i = N - 1; // Начинаем с самого младшего разряда (самого правого)

        // Пока текущий разряд "переполнен" (равен максимальной цифре N-1),
        // сбрасываем его в 0 и переходим к более старшему разряду (сдвигаемся влево)
        while (i >= 0 && queens[i] == N - 1) {
            queens[i] = 0;
            i--;
        }

        // Если мы переполнили самый старший разряд (вышли за левый край массива, i < 0),
        // значит счетовод прошел весь цикл. Счет окончен!
        if (i < 0) {
            break;
        }

        // Увеличиваем на 1 тот разряд, который остановил цикл while (не был переполнен)
        queens[i]++;
    }

    std::cout << "Найдено классических решений: " << solutionsCount << "\n";
    std::cout << "Проверено конфигураций в цикле (N^N): " << totalCombinations << "\n";
    // Для N=8: Решений 92. Комбинаций 8^8 = 16 777 216.

    return 0;
}
```

## Протокол прохождения (Механика переноса разряда)

Рассмотрим механику работы внутреннего цикла счетчика на примере маленькой доски 4x4 (`N=4`). Максимальная "цифра" в нашей позиционной 4-ичной системе — это `3`.

1. **Старт:** Массив заполнен нулями `[0, 0, 0, 0]`.
2. **Обычный инкремент:**
   - Индекс каретки `i=3`. Значение `queens[3]` равно `0` (не предел).
   - Вложенный цикл `while` не выполняется.
   - `queens[3]++`. Массив становится `[0, 0, 0, 1]`.
3. **Шаги со 2 по 4:** Массив циклично меняет последний элемент: `[..., 2]`, `[..., 3]`.
4. **Перенос разряда (Carry):** Массив сейчас принял вид `[0, 0, 0, 3]`. Мы снова пытаемся прибавить единицу.
   - `i=3`. Значение равно `3` (Переполнение!).
   - Заходим в цикл `while`. `queens[3]` сбрасывается в `0`. Индекс `i` становится `2`.
   - Значение `queens[2]` равно `0` (не предел). Цикл `while` завершается.
   - Срабатывает `queens[2]++`.
   - Итог каскадного шага: массив стал `[0, 0, 1, 0]`.
5. **Цепное ловинное переполнение:** Представим состояние массива `[0, 3, 3, 3]`:
   - Цикл `while` сбросит `queens[3]` в `0`, сдвинется на `i=2`.
   - Сбросит `queens[2]` в `0`, сдвинется на `i=1`.
   - Сбросит `queens[1]` в `0`, сдвинется на `i=0`.
   - Сработает `queens[0]++`.
   - Массив взрывным каскадом станет `[1, 0, 0, 0]`.
6. **Остановка программы:** Когда массив дойдет до абсолютного потолка `[3, 3, 3, 3]` и счетчик попытается прибавить единицу, цикл `while` сбросит все 4 ячейки в нули по цепочке, индекс `i` опустится до `-1`. Сработает терминальное условие выхода `if (i < 0) break;`.

## Архитектурное сравнение с Backtracking (Эффективность)

Подход на основе позиционных систем счисления (Brute-force) является **чудовищно неэффективным** по сравнению с рекурсивным поиском с возвратом (Backtracking).

- В рекурсивном **Бэктрекинге**, если мы на первом же столбце ставим двух ферзей в одну строку (получаем под-доску `[0, 0, X...X]`), функция проверки `isSafe` немедленно **отсекает** всю эту огромную ветвь как ошибочную. Процессор физически не будет перебирать остальные $N-2$ столбцов.
- В итеративном счетчике **$N$-ичной системы** алгоритм тупо, как механический калькулятор, переберет все числа от `[0, 0, 0, 0, 0, 0, 0, 0]` до `[0, 0, 7, 7, 7, 7, 7, 7]`. Он заставит АЛУ проверить каждую из этих 262 тысяч комбинаций в тяжелом цикле двойного перебора `isValidBoard`, потратив огромное количество тактов процессора, хотя _каждому_ очевидно с первого взгляда, что это тупик (первые два ферза стоят в строке 0 и бьют друг друга).

**Ресурсная оценка для задачи $N=8$:**

- Backtracking: Около `15 000` вызовов проверки безопасности позиций. Мгновенный результат.
- $k$-ричный счет: $8^8 = 16 777 216$ вызовов тяжелой функции проверки доски. (Хуже в тысячу раз).

**Инженерные плюсы метода:** Несмотря на просадку производительности, данный метод перебора не требует системного стека вызовов, не подвержен аварии `Stack Overflow` и не требует написания сложной древовидной архитектуры. Алгоритм пишется в один компактный цикл `while` и потребляет всего лишь $O(N)$ постоянной памяти ОЗУ.

## Резюме характеристик

| Элемент                      | Описание                                                                                                                                          |
| :--------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Парадигма**                | Итеративный полный перебор (Грубая сила / Brute-force) без эвристики и отсечений.                                                                 |
| **Математическая база**      | Алгоритмическая имитация каскадного счета в $k$-ричной позиционной системе счисления (где основание $k = N$).                                     |
| **Вычислительная сложность** | Временная сложность $O(N^N)$ переборов. При $N=8$ это около $16.7$ миллионов проверяемых матриц.                                                  |
| **Механика генерации шага**  | Поиск первого (справа налево) непереполненного разряда, обнуление всех элементов правее него и увеличение найденного разряда на $1$.              |
| **Главный недостаток**       | Из-за отсутствия раннего отсечения алгоритм выполняет гигантские блоки "холостой" работы, перебирая огромные пласты заведомо ошибочных вариантов. |
