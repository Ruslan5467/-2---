# Рекурсивные алгоритмы на примере вычисления факториала

## Теоретическая база: Рекурсия и Стек вызовов

В классическом программировании для реализации повторяющихся действий традиционно используются циклические конструкции (`for`, `while`). Однако в информатике и дискретной математике существует другой, более изящный (а зачастую и более мощный) способ решения повторяющихся задач — **Рекурсия**.

**Рекурсивный алгоритм** — это алгоритм, который в процессе выполнения обращается сам к себе. На уровне программного кода (в C++) это означает, что функция содержит инструкцию вызова самой себя (прямая рекурсия) или вызывает некоторую другую функцию, которая, в свою очередь, снова обращается к исходной (косвенная рекурсия).

Рекурсия основывается на принципе **«Разделяй и властвуй» (Divide and Conquer)**. Сложная (объемная) задача разбивается на точно такую же задачу, но слегка меньшего объема. Этот процесс дробления продолжается до тех пор, пока задача не станет настолько тривиальной, что её решение станет очевидным без дальнейших вычислений.

Для обеспечения работоспособности рекурсии архитектура процессора и операционной системы использует структуру данных, называемую **Стеком вызовов (Call Stack)**. Каждый рекурсивный вызов функции создает в этой памяти свой собственный изолированный "Фрейм вызова" со своими копиями локальных переменных и параметров.

### Обязательные компоненты рекурсивной функции

Любая грамотно спроектированная рекурсивная функция обязана состоять из двух фундаментальных частей, отсутствие любой из которых приведет алгоритм к катастрофе:

1.  **Базовый случай (Терминальное условие / Условие выхода):** Тривиальная ситуация, при которой функция возвращает конкретный результат немедленно, **без** вызова самой себя. Без терминального условия рекурсия становится бесконечной, что приводит к переполнению стека памяти (Stack Overflow) и аварийному схлопыванию приложения.
2.  **Рекурсивный шаг (Рекуррентное соотношение):** Инструкция, в которой функция вызывает саму себя, но обязательно с **измененными (уменьшенными) параметрами**, которые с каждым новым шагом неумолимо приближают алгоритм к Базовому случаю.

## Математическая модель Факториала

Классическим (хрестоматийным) примером для демонстрации принципов рекурсии является математическая операция вычисления факториала неотрицательного целого числа.

Факториал числа `n` (обозначается как `n!`) — это произведение всех натуральных чисел от 1 до `n`.
_Пример:_ `5! = 1 * 2 * 3 * 4 * 5 = 120`.

Эту же формулу можно записать через **рекуррентное математическое определение**:

1.  Утверждение (Базовый случай): `0! = 1`. _(Иногда добавляют `1! = 1` для оптимизации)_.
2.  Утверждение (Рекурсивный шаг): `n! = n * (n - 1)!`.

Математика прямо говорит: "Чтобы вычислить факториал 5, возьмите число 5 и умножьте его на факториал 4...". Это готовый словесный алгоритм.

## Пример использования (Реализация на C++)

Переведем рекуррентное математическое уравнение факториала непосредственно в машинный код.

```cpp
#include <iostream>

// Рекурсивная функция вычисления факториала
// Используем unsigned long long для предотвращения быстрого переполнения типа,
// так как факториал растет катастрофически быстро (уже 21! не влезет в 64-битное число).
unsigned long long factorial(unsigned int n) {
    // 1. БАЗОВЫЙ СЛУЧАЙ (Условие выхода из глубокого погружения)
    if (n == 0 || n == 1) {
        return 1;
    }

    // 2. РЕКУРСИВНЫЙ ШАГ
    // Функция ставит саму себя "на паузу" и ожидает результат от своего клона
    return n * factorial(n - 1);
}

int main() {
    std::cout << "Вычисление 4!: \n";
    unsigned long long result = factorial(4);

    std::cout << "\nРезультат: 4! = " << result << "\n";
    return 0;
}
```

## Протокол прохождения контрольного примера (Разбор Стэка)

Рассмотрим физику работы процессора при вычислении `factorial(4);`. Рекурсия всегда состоит из двух отчетливых фаз: **Фазы погружения (Спуска)** и **Фазы возврата (Подъема)**.

### Фаза 1. Погружение (Winding phase)

Функции вызывают друг друга, забивая оперативную память стека новыми фреймами (камерными хранилищами параметров). Вычисления (умножения) на этом этапе **не происходят**, так как каждая функция находится в ожидании ответа от следующей.

1.  **Вызов из main:** `factorial(4)`.
    - Проверка: `n == 0`? Нет.
    - Инструкция: Вернуть `4 * factorial(3)`. _Арифметико-логическое устройство процессора останавливается, так как правый операнд неизвестен._ Текущий фрейм (где n=4) "замораживается".
2.  **Глубина 1:** Вызывается клон `factorial(3)`. Выделяется новая память.
    - Проверка: `n == 0`? Нет.
    - Инструкция: Вернуть `3 * factorial(2)`. Заморозка.
3.  **Глубина 2:** Вызывается клон `factorial(2)`. Новая память.
    - Проверка: `n == 0`? Нет.
    - Инструкция: Вернуть `2 * factorial(1)`. Заморозка.
4.  **Глубина 3:** Вызывается клон `factorial(1)`.
    - Проверка: `n == 1`? **ДА! (Сработал Базовый Случай)**.
    - Инструкция: `return 1;`. Дно рекурсии достигнуто. Начинается возвращение "долгов".

### Фаза 2. Возврат и Схлопывание (Unwinding phase)

Когда достигнут Базовый случай, замороженные функции начинают последовательно "размораживаться", получать данные от вызванных ими клонов, выполнять умножение и уничтожать свои стековые фреймы, освобождая память.

1.  **Глубина 3:** Возвращает число `1`. Фрейм для `n=1` уничтожается.
2.  **Глубина 2:** Функция `factorial(2)` "размораживается", получив от клона число `1`.
    - Выполняет операцию `2 * 1 = 2`.
    - Возвращает число `2`. Фрейм для `n=2` уничтожается.
3.  **Глубина 1:** Функция `factorial(3)` получает число `2`.
    - Выполняет операцию `3 * 2 = 6`.
    - Возвращает число `6`. Фрейм для `n=3` уничтожается.
4.  **Оригинальный вызов:** Функция `factorial(4)` получает число `6`.
    - Выполняет финальную операцию `4 * 6 = 24`.
    - Возвращает число `24` обратно в функцию `main()`.

## Архитектурные риски: Переполнение Стека (Stack Overflow)

Память программного стека (Call Stack), в которой хранятся "замороженные" копии вызовов, аппаратно жестко ограничена (обычно 1-8 Мегабайт в зависимости от ОС).

Каждый рекурсивный вызов "собирает дань" с этой памяти — около 32-48 байт на создание нового фрейма. Если программист совершит ошибку (например, забудет написать Базовый случай `if (n==0)`) или попытается вычислить факториал ста тысяч с помощью рекурсии, процессор создаст сотни тысяч фреймов.
Свободная память стека закончится. Произойдет системное исключение доступа к памяти, и ядро ОС принудительно "убьет" программу ошибкой `Stack Overflow (Segmentation Fault)`.

Поэтому для тривиальных математических вычислений индустрия предпочитает использовать обычный итеративный (цикличный) подход (`for / while`), который использует всего **один** фрейм памяти, обходясь нулем накладных расходов.

## Резюме характеристик

| Характеристика             | Оценка / Описание                                                                  |
| :------------------------- | :--------------------------------------------------------------------------------- |
| **Суть механизма**         | Итеративное решение задачи путем вызова функцией самой себя                        |
| **Обязательные элементы**  | Базовый случай (Terminal condition) и Рекурсивный шаг с изменением данных          |
| **Аппаратное обеспечение** | Стек вызовов (Call Stack), хранящий состояние всех приостановленных клонов функции |
| **Типичные применения**    | Обход деревьев, графов, сортировки слиянием (Merge Sort/Quick Sort), фракталы      |
| **Главная угроза (Риск)**  | Фатальная архитектурная ошибка _Stack Overflow_ (переполнение системной памяти)    |
| **Альтернатива**           | Итеративный подход (циклы). Любую рекурсию можно математически развернуть в цикл.  |
